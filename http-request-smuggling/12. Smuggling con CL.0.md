# 📥 Smuggling con CL.0

## 📄 Descripción del laboratorio

Este laboratorio es vulnerable a **request smuggling tipo CL.0**, donde el servidor back-end ignora la cabecera **Content-Length** en determinados endpoints.

El objetivo es identificar una ruta vulnerable, introducir una solicitud hacia **/admin** y eliminar al usuario **carlos** desde el panel de administración.



## 📚 Teoría

En un ataque **CL.0**, el front-end respeta la cabecera `Content-Length`, pero el back-end la ignora completamente en algunas rutas.\
Esto provoca que el back-end procese todo el cuerpo de la petición como si fuera válido, permitiendo **inyectar una segunda petición HTTP** dentro del body de una POST aparentemente legítima.

Este comportamiento no suele afectar a toda la aplicación, sino a **endpoints concretos**, normalmente recursos estáticos o rutas mal configuradas.

La explotación consiste en encontrar uno de estos endpoints y aprovecharlo para acceder al panel de administración desde el back-end.



## 📝 Práctica

El **0** en CL.0 significa que el back-end **no tiene en cuenta Content-Length**, básicamente se traga todo pongas lo que le pongas.

Desde el front-end el acceso a **/admin** está bloqueado, pero si conseguimos que el back-end procese una petición directa, podremos saltarnos esta restricción.
