# 🎯 Rol de usuario controlado por cookie

## 📄 Descripción del laboratorio

Este laboratorio contiene un **panel de administración** disponible en:

```
/admin
```

El acceso a este panel se controla mediante una **cookie manipulable en el cliente**.

El objetivo es:

* Acceder al panel administrativo.
* Eliminar al usuario **carlos**.

Credenciales proporcionadas:

```
wiener:peter
```

 

## 📚 Teoría

Durante el proceso de autenticación, la aplicación establece una cookie llamada:

```
admin=false
```

Esta cookie se utiliza para determinar si el usuario tiene privilegios de administrador.

El problema es que el valor de la cookie:

* No está firmado.
* No está cifrado.
* No se valida en el servidor.

Por tanto, el cliente puede **modificar directamente el valor de la cookie**.

Si el valor se cambia a:

```
admin=true
```

el servidor considera al usuario como **administrador**, permitiendo acceso al panel `/admin`.

### 📌 Problemas de seguridad presentes

La aplicación presenta varios fallos de **Broken Access Control**:

* El rol del usuario se controla mediante un **parámetro manipulable por el cliente**.
* No existe **firma criptográfica** ni **hash** que proteja la cookie.
* El servidor **confía completamente en el valor recibido**.
* El panel administrativo no requiere **verificación adicional de permisos**.

Este tipo de vulnerabilidad es un ejemplo clásico de **privilege escalation mediante manipulación de cookies**.

 

## 📝 Práctica

### 🎯 Objetivo

Eliminar al usuario **carlos** desde el panel de administración.

 

### 1️⃣ Iniciar sesión

Accedemos a la aplicación utilizando las credenciales:

```
Usuario: wiener
Contraseña: peter
```

Una vez autenticados, revisamos las cookies establecidas por la aplicación.

<img width="356" height="88" alt="image" src="https://github.com/user-attachments/assets/b413ed3f-7b54-493f-9253-1eb24c8387a5" />
<br>

Observamos una cookie relevante:

```
admin=false
```

 

### 2️⃣ Manipulación de la cookie

Editamos la cookie desde:

* **DevTools del navegador**, o
* **Burp Suite**

Cambiamos su valor a:

```
admin=true
```

 

### 3️⃣ Acceso al panel de administración

Refrescamos la página o accedemos directamente a:

```
/admin
```

Resultado:

* El panel administrativo carga correctamente.
* El servidor ahora considera nuestro usuario como administrador.

 

### 4️⃣ Eliminación del usuario

Dentro del panel administrativo:

1. Localizamos al usuario **carlos**.
2. Pulsamos **Delete**.

La acción se ejecuta inmediatamente.
