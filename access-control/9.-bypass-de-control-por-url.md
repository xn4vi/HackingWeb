# 9. Bypass de Control por URL

## 🎯 Bypass de control por URL

### 📄 Descripción del laboratorio

Este laboratorio presenta un **bypass de control de acceso** provocado por una mala integración entre el **front-end** y el **back-end**.

Aunque el acceso directo al panel de administración está bloqueado, el servidor confía en una cabecera interna peligrosa:

```
X-Original-URL
```

El objetivo es:

* Acceder al **panel de administración**.
* Eliminar al usuario **carlos**.

***

### 📚 Teoría

La aplicación está dividida en dos capas:

#### 📌 Front-end (proxy / WAF / reverse proxy)

Esta capa filtra las rutas sensibles y bloquea el acceso directo a endpoints como:

```
/admin
```

#### 📌 Back-end (aplicación)

El backend soporta la cabecera:

```
X-Original-URL
```

Esta cabecera se utiliza normalmente para indicar la **ruta original de la petición** cuando el proxy realiza reescrituras internas.

#### 📌 Problema de seguridad

El fallo ocurre cuando:

* El **front-end valida únicamente la URL visible**.
* El **back-end confía en la cabecera `X-Original-URL`**.
* No existe validación cruzada entre ambas capas.

Esto permite que un atacante envíe una petición a una ruta permitida y, mediante la cabecera, indique al backend que procese una ruta diferente.

Por ejemplo:

* URL real enviada al servidor: `/`
* Ruta indicada al backend: `/admin`

El backend procesa la petición como si fuera administrativa.

***

### 📝 Práctica

#### 🎯 Objetivo

Acceder al panel `/admin` y eliminar al usuario **carlos**.

***

#### 1️⃣ Acceso directo bloqueado

Intentamos acceder directamente a:

```
/admin
```

Resultado:

* El acceso es bloqueado.
* El proxy devuelve **Access denied**.

Esto confirma que el **front-end está filtrando correctamente la ruta visible**.

<figure><img src="../../../.gitbook/assets/image (891).png" alt=""><figcaption></figcaption></figure>

***

#### 2️⃣ Uso de la cabecera `X-Original-URL`

El laboratorio nos indica que el backend soporta la cabecera **X-Original-URL**.

{% hint style="info" %}
La cabecera **X-Original-URL** es una cabecera HTTP que se utiliza para especificar la URL original de la solicitud del cliente.&#x20;
{% endhint %}

Interceptamos una petición válida, por ejemplo:

```http
GET /
```

Añadimos la cabecera:

```http
X-Original-URL: /admin
```

Enviamos la petición modificada.

<figure><img src="../../../.gitbook/assets/image (892).png" alt=""><figcaption></figcaption></figure>

***

#### 3️⃣ Acceso al panel administrativo

El backend procesa la petición utilizando el valor de la cabecera.

Resultado:

* Se ignora el bloqueo del front-end.
* La aplicación responde como si se hubiera solicitado `/admin`.
* Se carga el **panel de administración**.

***

#### 4️⃣ Eliminación del usuario carlos

Aplicamos la misma técnica para ejecutar la acción administrativa:

```http
X-Original-URL: /admin/delete?username=carlos
```

Enviamos la petición.

<figure><img src="../../../.gitbook/assets/image (893).png" alt=""><figcaption></figcaption></figure>

Resultado:

* El backend ejecuta la operación administrativa.
* El usuario **carlos** es eliminado.

***

#### 5️⃣ Resultado final

Se consigue:

* Bypassear el filtro del proxy.
* Acceder al panel administrativo sin autenticación.
* Ejecutar una acción destructiva en el sistema.

El laboratorio queda resuelto correctamente.
