# 🎯 Filtración de contraseña con ID por parámetro

## 📄 Descripción del laboratorio

Este laboratorio presenta una **escalada de privilegios horizontal** combinada con una **filtración crítica de credenciales**.

La página de cuenta de usuario expone la **contraseña actual dentro del HTML**, y además permite consultar el perfil de cualquier usuario manipulando un parámetro en la URL.

El objetivo es:

* Recuperar la **contraseña del usuario administrator**.
* Iniciar sesión con esa cuenta.
* Eliminar al usuario **carlos**.

Credenciales proporcionadas:

```
wiener:peter
```

 

## 📚 Teoría

La vulnerabilidad del laboratorio surge de la combinación de **dos fallos graves de seguridad**.

### 📌 Filtración de contraseña

La página `/my-account` muestra un formulario de edición del perfil del usuario autenticado.

Dentro de ese formulario aparece un campo como el siguiente:

```
<input type="password" name="password" value="PASSWORD_EN_TEXTO_PLANO">
```

Aunque el campo esté visualmente oculto por usar `type="password"`, la contraseña:

* Está presente en **texto plano dentro del HTML**.
* Se envía en la **respuesta HTTP del servidor**.
* Puede verse fácilmente inspeccionando el código de la página.

Esto constituye una vulnerabilidad crítica porque **las contraseñas nunca deberían enviarse al cliente**, ni siquiera ocultas visualmente.

### 📌 Control de acceso roto mediante parámetro `id`

La aplicación identifica al usuario únicamente mediante un parámetro en la URL:

```
/my-account?id=usuario
```

El servidor utiliza directamente ese valor para recuperar los datos del perfil.

No se verifica:

* Si el usuario autenticado es el propietario del perfil solicitado.
* Si tiene permiso para acceder a esa información.

Esto permite modificar el parámetro `id` y acceder al perfil de cualquier usuario.

 

## 📝 Práctica

### 🎯 Objetivo

Obtener la contraseña del **administrator**, iniciar sesión con esa cuenta y eliminar al usuario **carlos**.

 

### 1️⃣ Iniciar sesión

Accedemos a la aplicación con las credenciales:

```
Usuario: wiener
Contraseña: peter
```

Una vez autenticados, entramos en **My account**.

 

### 2️⃣ Confirmar la filtración de contraseña

En la página observamos el formulario de edición de perfil.

El campo **Password** aparece enmascarado en la interfaz.

<img width="389" height="105" alt="image" src="https://github.com/user-attachments/assets/0c22b4c6-e17f-4547-8b4e-c9d1c0991d80" />
<br>

Sin embargo, al inspeccionar el HTML de la página se observa algo similar a:

```html
<input type="password" name="password" value="peter">
```

Esto confirma que la contraseña del usuario **se encuentra en texto plano dentro del código HTML**.

<img width="768" height="172" alt="image" src="https://github.com/user-attachments/assets/7ae76d6f-ba6e-42c8-930a-194728ec93c0" />
<br>

 

### 3️⃣ Identificar el parámetro vulnerable

Observamos la URL de la página:

```
/my-account?id=wiener
```

Esto indica que el backend utiliza el parámetro `id` para identificar al usuario cuyo perfil debe mostrarse.

 

### 4️⃣ Acceder al perfil del administrador

Modificamos el parámetro en la URL:

```
/my-account?id=administrator
```

Recargamos la página.

 

### 5️⃣ Robo de credenciales

La página muestra ahora el perfil del usuario **administrator**.

Al inspeccionar el HTML encontramos el campo:

```html
<input type="password" name="password" value="PASSWORD_DEL_ADMIN">
```

Copiamos la **contraseña del administrador**.

<img width="768" height="564" alt="image" src="https://github.com/user-attachments/assets/1c34d408-e76f-4c04-81a6-4e85996a2ca8" />
<br>

### 6️⃣ Escalada vertical

Cerramos sesión y volvemos a iniciar sesión con:

```
Usuario: administrator
Contraseña: <contraseña_obtenida>
```

El inicio de sesión se realiza correctamente.

 

### 7️⃣ Eliminación del usuario carlos

Una vez autenticados como administrador:

1. Accedemos al panel de administración.
2. Buscamos al usuario **carlos**.
3. Pulsamos **Delete**.

La cuenta se elimina inmediatamente.

<br>
 

### 8️⃣ Resultado final

Se consigue:

* Obtener la contraseña del usuario **administrator**.
* Realizar **account takeover** de la cuenta administrativa.
* Escalar privilegios hasta administrador.
* Eliminar al usuario **carlos**.

El laboratorio queda resuelto correctamente.
