# 🌐 XSS reflejado en HTML sin codificación

## 📄 Descripción del laboratorio

Este laboratorio contiene una vulnerabilidad simple de **Cross-Site Scripting (XSS)** en la funcionalidad de búsqueda.

El valor introducido por el usuario se refleja directamente en la respuesta HTML **sin ningún tipo de codificación ni validación**.

🎯 **Objetivo del laboratorio:**

* Ejecutar la función `alert()` en el navegador mediante un ataque XSS.



## 📚 Teoría

Este laboratorio introduce el caso más básico de **Reflected Cross-Site Scripting (XSS)**.

Un XSS reflejado ocurre cuando:

```
1. El usuario envía un input a la aplicación
2. La aplicación incluye ese input en la respuesta HTML
3. El contenido se devuelve inmediatamente en la respuesta
4. No existe ningún tipo de codificación ni validación
```

En este caso concreto:

* El término de búsqueda se inserta directamente en el HTML.
* El navegador interpreta el contenido como **HTML o JavaScript**, no como texto.

Cuando el navegador procesa la respuesta:

```
1. Construye el DOM
2. Interpreta las etiquetas HTML
3. Ejecuta cualquier JavaScript encontrado
```

Aunque el laboratorio solo pide ejecutar `alert()`, un XSS real puede permitir:

```
robo de cookies
robo de tokens CSRF
keylogging
redirecciones maliciosas
ejecución de acciones en nombre del usuario
```



## 📝 Práctica

### 1️⃣ Identificar el punto vulnerable

Accedemos a la aplicación y localizamos el campo de búsqueda.

Introducimos un texto cualquiera y observamos que el término buscado aparece reflejado en la página de resultados.

Esto indica que el valor introducido por el usuario forma parte del HTML generado por el servidor.

Este comportamiento suele ser un primer indicador de **posible XSS reflejado**.



### 2️⃣ Probar inyección de HTML

Antes de intentar ejecutar JavaScript, probamos si el navegador interpreta HTML.

Introducimos en el campo de búsqueda:

```html
<h1>hola</h1>
```

Conclusión:

```
El input se refleja sin codificación HTML
```



### 3️⃣ Ejecutar JavaScript

Ahora probamos ejecutar JavaScript directamente.

Introducimos el payload:

```html
<script>alert(1)</script>
```

```
el JavaScript se ejecuta correctamente
existe un XSS reflejado funcional
el código se ejecuta en el contexto de la aplicación
```

En ese momento el laboratorio se marca como completado.



### 4️⃣ Resultado

Se consigue:

* Confirmar que el input se refleja sin codificación
* Inyectar código HTML
* Ejecutar JavaScript en el navegador

**Laboratorio resuelto.**
