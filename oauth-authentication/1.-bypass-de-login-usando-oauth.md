# 1. Bypass de Login usando OAuth

## 🔁 Bypass de login usando flujo implícito de OAuth

### 📄 **Descripción del laboratorio**

Este laboratorio utiliza **OAuth** para permitir a los usuarios iniciar sesión mediante una cuenta de redes sociales.

Debido a una validación incorrecta en la aplicación cliente, es posible iniciar sesión como otro usuario **sin conocer su contraseña**.

El objetivo es autenticarse como el usuario:

```
carlos
```

Email asociado:

```
carlos@carlos-montoya.net
```

Credenciales propias proporcionadas por el laboratorio:

```
wiener : peter
```

***

### 📚 Teoría

#### 📌 Flujo implícito de OAuth

En el **flujo implícito de OAuth**, el proceso de autenticación funciona de la siguiente manera:

1. El usuario se autentica en el proveedor OAuth.
2. El proveedor devuelve un **access\_token válido**.
3. El cliente envía ese token a su backend para completar el login.

En este laboratorio el cliente envía una petición que contiene:

```
email
username
access_token
```

#### 📌 Error de validación en la aplicación cliente

El problema aparece en la validación realizada por el backend.

El servidor:

* comprueba que el **access\_token sea válido**
* acepta los valores de **email** y **username** enviados por el cliente

Sin embargo, **no verifica que esos datos correspondan realmente al usuario asociado al token**.

Esto significa que un atacante puede:

* obtener un token válido para su propia cuenta
* reutilizar ese token
* modificar el email y username enviados al servidor

Como resultado, el backend crea una sesión para otro usuario.

Este tipo de fallo no es un problema del protocolo OAuth, sino de una **implementación incorrecta por parte de la aplicación cliente**.

***

### 📝 Práctica

#### 1️⃣ Realizar un login legítimo

Accedemos a:

```
My account
```

Seleccionamos iniciar sesión con OAuth.

Introducimos nuestras credenciales:

```
wiener : peter
```

<figure><img src="../../../.gitbook/assets/image (344).png" alt=""><figcaption></figcaption></figure>

Aceptamos los permisos solicitados por el proveedor OAuth.

La aplicación nos redirige de nuevo al laboratorio y quedamos autenticados como **wiener**

<figure><img src="../../../.gitbook/assets/image (345).png" alt=""><figcaption></figcaption></figure>

***

#### 2️⃣ Interceptar la petición final del flujo OAuth

Con **Burp Suite** interceptando el tráfico, localizamos la petición final que completa el login.

Petición observada:

```http
POST /authenticate
```

Parámetros típicos:

```
email=wiener@example.com
username=wiener
access_token=TOKEN_VALIDO
```

Esta es la petición vulnerable.

<figure><img src="../../../.gitbook/assets/image (346).png" alt=""><figcaption></figcaption></figure>

***

#### 3️⃣ Modificar la identidad del usuario

Enviamos la petición al **Repeater**.

Modificamos los parámetros del usuario objetivo:

```
email=carlos@carlos-montoya.net
username=carlos
access_token=TOKEN_VALIDO
```

El `access_token` sigue siendo válido, ya que pertenece a nuestra sesión OAuth.

La única modificación realizada es la identidad enviada al backend.

<figure><img src="../../../.gitbook/assets/image (347).png" alt=""><figcaption></figcaption></figure>

***

#### 4️⃣ Enviar la petición manipulada

Enviamos la petición modificada.

El servidor responde correctamente y devuelve una **cookie de sesión válida**.

No se produce ningún error ni validación adicional.

Esto indica que el backend ha aceptado los datos manipulados.

***

#### 5️⃣ Acceder a la cuenta de Carlos

Utilizamos la opción **Show response in browser** o copiamos la cookie de sesión generada.

Abrimos la respuesta en el navegador.

La aplicación muestra la sesión autenticada como:

```
carlos
```

<figure><img src="../../../.gitbook/assets/image (348).png" alt=""><figcaption></figcaption></figure>

***

#### 6️⃣ Resultado

Mediante un fallo en la validación del flujo implícito de OAuth se ha conseguido:

* reutilizar un **access\_token válido**
* modificar la identidad enviada al backend
* iniciar sesión como **carlos**
* realizar un **Account Takeover sin conocer la contraseña**

El laboratorio queda resuelto.
