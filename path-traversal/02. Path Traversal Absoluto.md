# 📂 Path traversal - bypass absoluto con secuencias bloqueadas

## 📄 Descripción del laboratorio

Este laboratorio contiene una vulnerabilidad de **Path Traversal** en la funcionalidad de visualización de imágenes de productos.

La aplicación intenta protegerse filtrando las secuencias clásicas de retroceso (`../`), pero no valida ni restringe el uso de **rutas absolutas desde la raíz del sistema de archivos**.

El objetivo es recuperar el contenido del archivo:

```
/etc/passwd
```

especificando directamente una **ruta absoluta**.

 

## 📚 Teoría

En este laboratorio, la aplicación construye la ruta del archivo a partir del parámetro `filename` y aplica un filtro básico que bloquea cualquier intento de usar secuencias `../`.

El error está en asumir que bloquear rutas relativas es suficiente.

Cuando el servidor recibe un valor que comienza con `/`, el sistema operativo lo interpreta como una **ruta absoluta**, ignorando completamente cualquier directorio base previamente concatenado.

Esto provoca que el prefijo considerado “seguro” (por ejemplo `/var/www/images/`) quede inutilizado.

La vulnerabilidad existe debido a:

* Filtrado incompleto (solo bloquea `../`)
* Falta de validación de rutas absolutas (`/etc/passwd`)
* Concatenación directa del parámetro `filename`
* Devolución directa del contenido del archivo en la respuesta HTTP

Como resultado, es posible realizar **lectura arbitraria de archivos usando rutas absolutas**, incluso cuando existen filtros activos.

 

## 📝 Práctica

### 🎯 Objetivo

Leer el archivo:

```
/etc/passwd
```

 

### 1️⃣ Identificación de la petición vulnerable

Se navega por la tienda y se selecciona cualquier producto.

Se observa la petición que carga la imagen del producto:

```http
GET /image?filename=6.jpg
```

La petición se intercepta con **Burp Suite** y se envía al **Repeater**.

 

### 2️⃣ Prueba con path traversal clásico

Se prueba primero el payload clásico:

```
/image?filename=../../../../../../etc/passwd
```


### 4️⃣ Resultado

La respuesta HTTP contiene directamente el contenido del archivo:

```
/etc/passwd
```

Esto confirma que:

* El filtro solo actúa sobre **rutas relativas**.
* Las **rutas absolutas no están validadas**.
* El backend permite acceder a **archivos arbitrarios** si se indica la ruta completa desde `/`.
