# 1. Path Traversal Simple

## 📂 Path traversal - caso simple

### 📄 Descripción del laboratorio

Este laboratorio contiene una vulnerabilidad de **Path Traversal** (también conocida como **Directory Traversal**) en la funcionalidad de visualización de imágenes de productos.

La aplicación construye la ruta del archivo a partir del parámetro `filename` sin aplicar ninguna validación ni sanitización.

El objetivo es **recuperar el contenido del archivo**:

```
/etc/passwd
```

explotando esta vulnerabilidad.

***

### 📚 Teoría

En este laboratorio, la aplicación carga imágenes de productos mediante una petición **GET** que incluye el parámetro `filename`.

Internamente, el servidor concatena este valor a una ruta base del sistema de archivos, algo similar a:

```
/var/www/images/ + filename
```

El problema es que no existe ninguna protección contra secuencias de retroceso (`../`), lo que permite:

* Escapar del directorio `/images/`
* Subir por la jerarquía de directorios
* Acceder a cualquier archivo del sistema al que el proceso tenga permisos de lectura

Esto hace posible apuntar directamente a archivos sensibles como:

```
/etc/passwd
```

La vulnerabilidad existe debido a:

* Concatenación directa del parámetro `filename`
* Ausencia de filtrado de `../`
* Falta de una lista blanca de archivos permitidos
* Devolución directa del contenido del archivo en la respuesta HTTP

Como resultado, es posible realizar **lectura arbitraria de archivos del sistema**.

***

### 📝 Práctica

#### 🎯 Objetivo

Leer el archivo:

```
/etc/passwd
```

***

#### 1️⃣ Identificación de la petición vulnerable

Se navega por la tienda y se selecciona cualquier producto.

Observamos la petición que carga la imagen del producto:

```http
GET /image?filename=6.jpg
```

<figure><img src="../../../.gitbook/assets/image (606).png" alt=""><figcaption></figcaption></figure>

Se intercepta la petición con **Burp Suite** y se envía al **Repeater** para analizarla.

***

#### 2️⃣ Análisis de la ruta del servidor

Es probable que la ruta real del servidor se construya de forma similar a:

```
/var/www/html/images/6.jpg
```

Como no existe sanitización, es posible utilizar secuencias `../` para salir del directorio `/images/` y alcanzar la raíz del sistema.

***

#### 3️⃣ Explotación de Path Traversal

Se prueba el siguiente payload:

```
/image?filename=../../../../../../etc/passwd
```

Se envía la petición modificada.

<figure><img src="../../../.gitbook/assets/image (607).png" alt=""><figcaption></figcaption></figure>

***

#### 4️⃣ Resultado

La respuesta HTTP contiene directamente el contenido del archivo:

```
/etc/passwd
```

Esto confirma que:

* Se ha escapado correctamente del directorio base.
* Es posible leer archivos arbitrarios del sistema.

{% hint style="info" %}
Si se utilizan más secuencias `../` de las necesarias, no ocurre ningún problema.\
El sistema simplemente ignora los retrocesos adicionales.
{% endhint %}
