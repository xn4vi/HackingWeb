# 2. Bypass de 2FA

## 🔓 Bypass básico de 2FA

### 📄 Descripción del laboratorio

El laboratorio implementa **autenticación de dos factores (2FA)**, pero de forma defectuosa.

Aunque el sistema solicita un **código 2FA** tras validar usuario y contraseña, no protege correctamente las rutas sensibles, lo que permite acceder directamente a la cuenta sin completar el segundo factor.

🎯 **Objetivo del laboratorio:**

* Acceder a la página de cuenta de **Carlos** sin conocer su código 2FA

Credenciales disponibles:

```
wiener:peter
carlos:montoya
```

***

### 📚 Teoría

Este laboratorio muestra un error común en implementaciones de **2FA**: tratar el segundo factor como una pantalla, en lugar de un **estado de autenticación**.

El flujo correcto debería ser:

1. El usuario introduce **credenciales válidas**
2. El servidor marca la sesión como **parcialmente autenticada**
3. Se solicita el **código 2FA**
4. Solo tras validarlo, la sesión pasa a estar **completamente autenticada**
5. Las rutas sensibles verifican este estado antes de responder

El problema aparece cuando el backend **no comprueba en cada endpoint protegido** si el usuario ha completado realmente el segundo factor.

En ese caso, basta con:

* Tener **usuario y contraseña válidos**
* Evitar la pantalla de **2FA**
* Navegar directamente a una ruta protegida como `/my-account`

El servidor asume erróneamente que cualquier usuario con sesión iniciada ya ha completado el segundo factor.

***

### 📝 Práctica

#### 1️⃣ Identificación de la ruta protegida

Iniciamos sesión con nuestras credenciales:

```
wiener:peter
```

Después de introducir usuario y contraseña, el sistema solicita el **código 2FA** que recibimos por email.

Introducimos el código correctamente y accedemos a la página de cuenta.

Observamos la URL a la que somos redirigidos:

```
/my-account
```

***

#### 2️⃣ Inicio de sesión con la víctima

Cerramos sesión.

Ahora iniciamos sesión con las credenciales de la víctima:

```
carlos:montoya
```

El sistema valida el usuario y la contraseña y nos muestra la pantalla para introducir el **código 2FA**, que no conocemos.

***

#### 3️⃣ Acceso directo a la cuenta

En lugar de introducir el código, cambiamos manualmente la URL en el navegador y accedemos directamente a:

```
/my-account
```

La página se carga correctamente sin solicitar el código 2FA.

<figure><img src="../../../.gitbook/assets/image (154).png" alt=""><figcaption></figcaption></figure>

***

#### 4️⃣ Resultado

Se consigue:

* Acceder a la cuenta de **Carlos**
* Evitar completamente la verificación **2FA**

El servidor no valida que el segundo factor haya sido completado antes de servir contenido sensible.

✅ **Laboratorio resuelto.**
