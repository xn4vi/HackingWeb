# 2. SSRF Básico contra Sistema Interno

## 🌐 SSRF básico contra sistema interno

### 📄 Descripción del laboratorio

Este laboratorio presenta una vulnerabilidad de **Server-Side Request Forgery (SSRF)** en la funcionalidad de comprobación de existencias.

A diferencia del laboratorio anterior, el servicio administrativo **no está en localhost**, sino en otra máquina dentro de la red interna.

🎯 **Objetivo del laboratorio:**

* Usar SSRF para escanear el rango interno `192.168.0.0/24`
* Localizar la interfaz de administración en el puerto `8080`
* Eliminar al usuario `carlos`

***

### 📚 Teoría

En este escenario el SSRF es **parcialmente ciego (blind SSRF)**.

Características:

* ❌ No vemos directamente la respuesta completa del servicio interno
* ✅ El servidor vulnerable sí realiza la petición HTTP al destino indicado

Solo recibimos **respuestas indirectas**, como:

* Mensajes de error
* Timeouts
* Diferencias en longitud de respuesta
* Códigos HTTP

Detalles del laboratorio:

El panel de administración:

* ❌ No está en `localhost`
* ✅ Está en otra máquina de la red privada `192.168.0.0/24`
* 🔌 Escucha en el puerto `8080`

Este servicio **no es accesible desde Internet**, únicamente desde la red interna.

Usando SSRF podemos convertir el servidor vulnerable en un **proxy de escaneo interno**.

La idea es:

1. Detectar qué IP interna tiene un servidor web en `:8080`
2. Acceder a `/admin`
3. Ejecutar la acción para eliminar a `carlos`

***

### 📝 Práctica

#### 1️⃣ Punto de entrada SSRF

Abrimos cualquier producto y utilizamos **Check stock**.

Interceptamos la petición con **Burp Suite**.

Ejemplo de petición:

```http
POST /product/stock HTTP/1.1
Host: ID-DEL-LAB.web-security-academy.net
Content-Type: application/x-www-form-urlencoded

stockApi=http://stock.weliketoshop.net:8080/product/stock/check?productId=1&storeId=1
```

Enviamos la petición al **Repeater** para realizar pruebas.

<figure><img src="../../../.gitbook/assets/image (114).png" alt=""><figcaption></figcaption></figure>

***

#### 2️⃣ Confirmando que no está en localhost

Probamos primero una URL local:

```http
stockApi=http://localhost:8080/admin
```

Resultado:

Timeout o error.

Esto indica que el panel **no está en la misma máquina que el servidor web del laboratorio**.

Conclusión:

El panel administrativo está en **otra IP de la red interna**.

***

#### 3️⃣ Escaneo de la red interna con Intruder

Sabemos:

* Red interna: `192.168.0.0/24`
* Puerto: `8080`

Usamos **Burp Intruder** para escanear.

Configuración:

Positions\
Marcamos únicamente el último octeto:

```http
stockApi=http://192.168.0.§1§:8080
```

Payloads

* Tipo: Numbers
* From: 1
* To: 255
* Step: 1

Importante:

Desmarcar **URL-encode these characters** para que `:` y `/` no se codifiquen.

<figure><img src="../../../.gitbook/assets/image (117).png" alt=""><figcaption></figcaption></figure>

***

#### 4️⃣ Analizando resultados

Ejecutamos el ataque.

La mayoría de respuestas muestran:

* Timeout
* Connection refused
* Respuestas vacías

Una IP destaca, por ejemplo:

```
192.168.0.81:8080 → 404 Not Found
```

<figure><img src="../../../.gitbook/assets/image (116).png" alt=""><figcaption></figcaption></figure>

Un **404** indica que hay **un servidor web activo**, aunque la ruta solicitada no exista.

***

#### 5️⃣ Acceso al panel admin interno

Probamos la IP detectada en **Repeater**:

```http
stockApi=http://192.168.0.81:8080/admin
```

<figure><img src="../../../.gitbook/assets/image (118).png" alt=""><figcaption></figcaption></figure>

La respuesta devuelve el **HTML del panel de administración**, incluyendo:

* Listado de usuarios
* Acciones administrativas

***

#### 6️⃣ Eliminando al usuario carlos

Probamos la acción para eliminar al usuario:

```http
stockApi=http://192.168.0.81:8080/admin/delete?username=carlos
```

Enviamos la petición.

<figure><img src="../../../.gitbook/assets/image (119).png" alt=""><figcaption></figcaption></figure>

***

#### 7️⃣ Resultado final

La respuesta devuelve un **302 Found** o un mensaje de éxito.

El servidor interno ejecuta la acción administrativa.

El usuario **carlos ha sido eliminado**.

✅ **Laboratorio resuelto.**
