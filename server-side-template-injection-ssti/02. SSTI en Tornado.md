# 💉 Inyección de plantillas del lado servidor (SSTI) en Tornado

## 📄 Descripción del laboratorio

La aplicación permite a los usuarios personalizar cómo se muestra su nombre en los comentarios (**nickname o nombre real**).

Este valor se inserta directamente dentro de una **plantilla Tornado**, sin sanitización ni escape previo.

Tornado evalúa las expresiones `{{ }}` como **código Python**, por lo que al controlar parte de esa expresión es posible romper el contexto y ejecutar **código Python arbitrario en el servidor**.

El objetivo es:

* Ejecutar **código Python arbitrario** mediante SSTI.
* Eliminar el archivo:

```
/home/carlos/morale.txt
```

 

## 📚 Teoría

**Server-Side Template Injection (SSTI)** ocurre cuando input controlado por el usuario se interpola directamente en una plantilla que se evalúa en el servidor.

En **Tornado**, el motor de plantillas utiliza la siguiente sintaxis:

```python
{{ expresión_python }}
```

Estas expresiones:

* Se evalúan como **Python real**.
* Tienen acceso a:
  * **Builtins** (`__import__`, `eval`, etc.).
  * **Módulos estándar** (`os`, `subprocess`).
  * **Objetos del contexto del template**.

Si la aplicación construye algo como:

```python
{{ user.display_name }}
```

y el valor de `display_name` es controlado por el usuario, es posible:

* Cerrar la expresión con `}}`.
* Inyectar **código Python**.
* Abrir una nueva expresión con `{{` si es necesario.

Esto permite **ejecución remota de código (RCE)** en el servidor.

 

## 📝 Práctica

### 🎯 Objetivo

Eliminar el archivo:

```
/home/carlos/morale.txt
```

mediante **SSTI en Tornado**.

 

### 1️⃣ Identificación de SSTI

Se inicia sesión con:

```
wiener:peter
```

Se accede a la sección de **perfil**, donde es posible elegir cómo se muestra el nombre en los comentarios.

Ese valor se refleja dinámicamente, por ejemplo:

```
Posted by: wiener
```

Este comportamiento resulta sospechoso porque el contenido:

* Es **dinámico**.
* Depende directamente de **input del usuario**.
* Se **renderiza del lado servidor**.

 

### 2️⃣ Detección del motor de plantillas

### 3️⃣ Ejecución de código Python

En Tornado es posible importar módulos y ejecutar comandos del sistema.

Se sabe que:

```python
os.system()
```

permite ejecutar comandos del sistema operativo.

Se construye un payload que:

* Cierra la expresión original.
* Importa el módulo `os`.
* Ejecuta el comando `rm`.

Payload final (URL-encoded donde aplica):

```python
user.name}}{%25+import+os+%25}{{os.system('rm%20/home/carlos/morale.txt')
```

Este payload:

* Rompe el **contexto del template**.
* Inyecta **código Python real**.
* Ejecuta el comando en el servidor.
