# 1. CI Simple

## 💉 Inyección de comandos, caso simple

### 📄 Descripción del laboratorio

La funcionalidad **Check stock** ejecuta un comando del sistema operativo que incluye directamente los parámetros `productId` y `storeId` enviados por el usuario.

No existe ningún tipo de **sanitización, validación ni escaping**, lo que permite inyectar comandos arbitrarios.

🎯 **Objetivo del laboratorio:**

* Ejecutar el comando `whoami`
* Identificar el usuario del sistema bajo el que se ejecuta el proceso del servidor

***

### 📚 Teoría

La **OS Command Injection** ocurre cuando una aplicación construye comandos del sistema concatenando entrada controlada por el usuario.

Por ejemplo:

```
check_stock <productId> <storeId>
```

Si la aplicación hace algo como:

```bash
system("check_stock " + productId + " " + storeId)
```

Sin filtros, podemos inyectar separadores de comandos:

* `;` → ejecuta el siguiente comando siempre
* `&&` → ejecuta si el anterior fue exitoso
* `||` → ejecuta si el anterior falla
* `|` → pipe de salida

En escenarios **non-blind**, la salida del comando se devuelve directamente en la respuesta HTTP, facilitando la confirmación visual del ataque.

***

### 📝 Práctica

#### 1️⃣ Interceptar la petición

Navegamos por la tienda.

Seleccionamos cualquier producto y pulsamos **Check stock**.

Interceptamos la petición **POST** con **Burp Suite**.

<figure><img src="../../../.gitbook/assets/image (618).png" alt=""><figcaption></figcaption></figure>

Cuerpo original:

```
productId=1&storeId=1
```

***

#### 2️⃣ Inyección del comando

Modificamos el parámetro `storeId` añadiendo un separador `;` y el comando deseado:

```bash
productId=1&storeId=1;whoami
```

El comando que se ejecuta internamente queda equivalente a:

```bash
check_stock 1 1; whoami
```

***

#### 3️⃣ Ejecución

Enviamos la petición modificada.

El servidor ejecuta ambos comandos en secuencia y la salida completa se devuelve en la respuesta HTTP.

<figure><img src="../../../.gitbook/assets/image (619).png" alt=""><figcaption></figcaption></figure>

***

#### 4️⃣ Resultado final

En la respuesta vemos la salida de `whoami`, por ejemplo:

```
www-data
```

Esto confirma que:

* Existe **inyección de comandos**
* El proceso se ejecuta como un usuario del servidor web (ej. `www-data`, `apache`, etc.)

✅ **Laboratorio resuelto.**
