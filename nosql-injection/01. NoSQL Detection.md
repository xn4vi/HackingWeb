# 💉 Detección de inyección NoSQL

## 📄 Descripción del laboratorio

El laboratorio implementa un **filtro de categorías** que consulta una base de datos **MongoDB**.

La aplicación construye la consulta **concatenando directamente la entrada del usuario**, sin sanitización ni validación.

🎯 **Objetivo del laboratorio:**

* Realizar una **inyección NoSQL**
* Forzar a la aplicación a mostrar **todos los productos**
* Incluir los que están marcados como **unreleased**

 

## 📚 Teoría

Las **inyecciones NoSQL** aparecen cuando una aplicación genera consultas dinámicas a bases de datos NoSQL (como **MongoDB**) usando directamente datos controlados por el usuario.

Ejemplo vulnerable:

```javascript
db.products.find({ category: req.query.category })
```

Otro patrón vulnerable es el uso de `$where` con JavaScript:

```javascript
db.products.find({
  $where: "this.category == '" + req.query.category + "'"
})
```

En estos casos:

* La entrada del usuario se evalúa como **código JavaScript**
* Se pueden inyectar operadores lógicos como `||`, `&&`, `true`, `1`
* El objetivo es construir una condición que **siempre evalúe a verdadero**

Indicadores típicos de **NoSQL Injection**:

* Errores de sintaxis JavaScript
* Mensajes que mencionan **MozJS** (motor JavaScript de MongoDB)
* Comportamientos alterados en filtros o consultas

Estas técnicas se utilizan para:

* Detectar **NoSQL Injection**
* Enumerar datos
* Evadir filtros o controles de acceso

 

## 📝 Práctica

### 1️⃣ Análisis del filtro

Navegamos por la tienda y cambiamos entre distintas categorías.

Interceptamos la petición que contiene el parámetro `category`.

Probamos valores inválidos:

```
'
"
{}
```

<br>
La aplicación devuelve un **error de sintaxis JavaScript** y menciona **MozJS**.

Esto confirma que la entrada del usuario **se evalúa como código JavaScript**, lo que indica una posible **NoSQL Injection**.

 

### 2️⃣ Construcción del payload

Buscamos una condición que siempre sea verdadera.

Payload utilizado:

```
Pets' || true || '
```

La consulta resultante se interpreta como:

```
this.category == 'Pets' || true || ''
```

Como `true` siempre se evalúa como verdadero, la condición se cumple para **todos los documentos**.

 

### 3️⃣ Ejecución
