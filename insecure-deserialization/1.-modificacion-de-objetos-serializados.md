# 1. Modificación de Objetos Serializados

## 🧬 Modificación de objetos serializados

### 📄 Descripción del laboratorio

Este laboratorio utiliza un mecanismo de sesión basado en **objetos PHP serializados** almacenados en una cookie.

El backend deserializa directamente el contenido de la cookie para reconstruir el estado de la sesión del usuario.\
Esto permite modificar manualmente el objeto serializado y alterar atributos sensibles.

🎯 **Objetivo del laboratorio:**

* Escalar privilegios modificando el objeto de sesión
* Obtener acceso administrativo
* Eliminar al usuario `carlos`

Credenciales iniciales:

```
wiener : peter
```

***

### 📚 Teoría

La aplicación implementa el estado de sesión mediante **serialización de objetos PHP**.

El flujo interno es el siguiente:

1. El usuario inicia sesión.
2. El backend crea un objeto PHP que representa la sesión.
3. El objeto se serializa con `serialize()`.
4. El resultado se codifica (normalmente Base64).
5. El valor se guarda en una cookie.

En cada petición posterior el servidor:

```
lee la cookie
la decodifica
ejecuta unserialize()
confía en los valores del objeto
```

Un ejemplo del objeto serializado sería:

```
O:4:"User":2:{s:8:"username";s:6:"wiener";s:5:"admin";b:0;}
```

Desglose:

```
O:4:"User"     → objeto de la clase User
2:{...}       → número de atributos
s:8:"username";s:6:"wiener"; → nombre de usuario
s:5:"admin";b:0;             → flag de administrador
```

El problema crítico es que:

```
la cookie no está firmada
no hay verificación de integridad
el backend confía en el contenido deserializado
```

Esto permite modificar el objeto manualmente y cambiar el atributo:

```
b:0 → b:1
```

Con ello se obtiene privilegios administrativos.

***

### 📝 Práctica

#### 1️⃣ Iniciar sesión y capturar la cookie

Iniciamos sesión con:

```
wiener : peter
```

Accedemos a **/my-account** y observamos la cookie de sesión en Burp.

Ejemplo:

```
session=eyJ...==
```

<figure><img src="../../../.gitbook/assets/image (1088).png" alt=""><figcaption></figcaption></figure>

***

#### 2️⃣ Decodificar la cookie

Seleccionamos el valor de la cookie y lo enviamos a **Burp Decoder**.

Proceso habitual:

```
URL decode
Base64 decode
```

Tras decodificar aparece el objeto PHP serializado:

```
O:4:"User":2:{s:8:"username";s:6:"wiener";s:5:"admin";b:0;}
```

<figure><img src="../../../.gitbook/assets/image (1089).png" alt=""><figcaption></figcaption></figure>

***

#### 3️⃣ Modificar el objeto serializado

Localizamos el atributo:

```
s:5:"admin";b:0;
```

Lo cambiamos por:

```
s:5:"admin";b:1;
```

El objeto modificado queda así:

```
O:4:"User":2:{s:8:"username";s:6:"wiener";s:5:"admin";b:1;}
```

No se modifican longitudes ni estructura del objeto.

***

#### 4️⃣ Recodificar la cookie

El objeto modificado debe volver a codificarse:

```
Base64 encode
URL encode (si aplica)
```

En Burp puede hacerse desde el **Inspector**, aplicando los cambios automáticamente.

La cookie queda actualizada en la petición.

<figure><img src="../../../.gitbook/assets/image (1090).png" alt=""><figcaption></figcaption></figure>

***

#### 5️⃣ Confirmar privilegios de administrador

Enviamos la petición con la cookie modificada.

La aplicación ahora muestra funcionalidades administrativas.

Accedemos manualmente a:

```
/admin
```

<figure><img src="../../../.gitbook/assets/image (1091).png" alt=""><figcaption></figcaption></figure>

El acceso se concede correctamente.

<figure><img src="../../../.gitbook/assets/image (1092).png" alt=""><figcaption></figcaption></figure>

***

#### 6️⃣ Eliminar al usuario carlos

Desde el panel administrativo localizamos al usuario `carlos`.

La acción administrativa ejecuta una petición similar a:

```
/admin/delete?username=carlos
```

El servidor procesa la petición correctamente y elimina la cuenta.

<figure><img src="../../../.gitbook/assets/image (1093).png" alt=""><figcaption></figcaption></figure>

***

#### 7️⃣ Resultado

Se consigue:

* Escalar privilegios modificando un objeto serializado
* Obtener acceso administrativo
* Eliminar al usuario `carlos`

✅ **Laboratorio resuelto.**
