# 🧑‍💻 Fallo lógico de bajo nivel (integer overflow)

## 📄 Descripción del laboratorio

Este laboratorio presenta una vulnerabilidad de **integer overflow** en la lógica de cálculo del carrito.

Al añadir una cantidad muy elevada de productos, el total supera el valor máximo permitido por el tipo de dato entero del backend, provocando un desbordamiento.

El objetivo es:

* Manipular el valor total del carrito mediante overflow
* Ajustarlo a un valor dentro del crédito disponible
* Completar la compra del producto



## 📚 Teoría

Algunos lenguajes de programación utilizan tipos de datos enteros con un límite máximo.

Un entero típico de 32 bits tiene como valor máximo:

```
2,147,483,647
```

### 📌El fallo

Cuando el valor supera ese límite:

* Se produce un **overflow**
* El número pasa al rango negativo

Ejemplo conceptual:

```
MAX_INT + 1 → valor negativo
```

Esto provoca que:

* El precio total del carrito se vuelva negativo
* El sistema continúe operando con ese valor incorrecto

### 📌 Impacto

Esto permite:

* Manipular el total del carrito
* Reducir precios de forma artificial
* Forzar valores dentro del crédito disponible
