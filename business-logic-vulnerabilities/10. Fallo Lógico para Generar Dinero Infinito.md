# 🧑‍💻 Fallo lógico para generar dinero infinito

## 📄 Descripción del laboratorio

Este laboratorio presenta una vulnerabilidad en la lógica de negocio que permite **generar crédito de forma indefinida**.

El sistema permite comprar tarjetas regalo (gift cards) con descuento y posteriormente canjearlas por su valor completo, generando un beneficio en cada iteración.

El objetivo es:

* Generar crédito ilimitado en la cuenta
* Utilizar ese saldo para comprar productos caros



## 📚 Teoría

Las aplicaciones deben garantizar que:

* No se pueda obtener más valor del que se paga
* Los descuentos no generen beneficios directos

### 📌 El fallo

El sistema permite:

* Comprar gift cards con un cupón de descuento
* Canjearlas por su valor completo

Ejemplo:

```
Compra: 10€ → con descuento → 7€
Canje: +10€
Beneficio: +3€
```

Además:

* El cupón `SIGNUP30` no caduca
* Puede reutilizarse indefinidamente

### 📌 Impacto

Esto permite:

* Generar dinero infinito
* Romper completamente la lógica económica
* Comprar cualquier producto sin limitaciones



## 📝 Práctica

### 1️⃣ Identificar el vector

Nos logueamos en la aplicación.

Observamos:

* Existe funcionalidad de gift cards
* Hay un producto tipo gift card en la tienda

![](../images/blg33.png)
<br>

También encontramos un cupón al registrarnos en la newsletter:

```
SIGNUP30
```

![](../images/blg34.png)
<br>
![](../images/blg35.png)



### 2️⃣ Confirmar el fallo

Compramos una gift card.

Aplicamos el cupón.

![](../images/blg36.png)
<br>

Observamos:

* Precio de compra: 7€
* Valor al canjear: 10€

Resultado:

* Ganancia de 3€

![](../images/blg37.png)
<br>

El cupón no caduca y puede reutilizarse.



### 3️⃣ Preparar la automatización

Realizamos el flujo manual completo desde Burp para registrar todas las peticiones en el historial.

Luego:

```
Settings → Sessions → Add → Run a macro
```

![](../images/blg38.png)
<br>

Añadimos las peticiones necesarias (manteniendo Ctrl para seleccionarlas todas).

![](../images/blg39.png)
<br>
![](../images/blg40.png)
<br>
![](../images/blg41.png)



### 4️⃣ Configurar la macro

Seleccionamos la petición donde aparece el código de la gift card.

Entramos en:

```
Configure item
```

![](../images/blg42.png)
<br>

Añadimos un parámetro:

* Nombre del parámetro
* Seleccionamos en la respuesta el valor de la gift card

![](../images/blg43.png)
<br>

Esto permite capturar dinámicamente el código generado.



### 5️⃣ Reutilizar el valor dinámico

En la siguiente petición (donde se canjea la gift card):

* Repetimos el proceso
* Sustituimos el valor por el capturado anteriormente

![](../images/blg44.png)
<br>

![](../images/blg45.png)
<br>

De este modo:

* Cada ejecución usa una nueva gift card válida



### 6️⃣ Probar la macro

Ejecutamos:

```
Test macro
```

Verificamos que:

* El saldo de la cuenta aumenta correctamente



### 7️⃣ Activar la automatización

Guardamos la macro.

En:

```
Scope → Include all URLs
```

![](../images/blg46.png)


### 8️⃣ Ejecutar el ataque

Interceptamos cualquier petición (por ejemplo en home) y la enviamos a Repeater.

Cada vez que enviamos la request:

* Se ejecuta la macro
* El saldo aumenta

Para automatizar:

* Enviamos la petición a Intruder
* Payload: null payload
* Número de peticiones: por ejemplo 100

Configuramos:

```
Resource pool → 1 request at a time
```

Lanzamos el ataque.



### 9️⃣ Resultado del ataque

El saldo de la cuenta:

* Aumenta automáticamente
* Se multiplica con cada ejecución
