# 🧑‍💻 Validación insuficiente del flujo de trabajo

## 📄 Descripción del laboratorio

Este laboratorio presenta una vulnerabilidad en la lógica del proceso de compra, donde el flujo de validación no se aplica correctamente.

El sistema valida la compra únicamente en un punto concreto del proceso, permitiendo reutilizar esa validación en contextos no previstos.

El objetivo es:

* Comprar la **Lightweight l33t leather jacket** sin crédito suficiente
* Aprovechando una validación incorrecta del flujo



## 📚 Teoría

Las aplicaciones deben validar correctamente cada paso del flujo de negocio, especialmente en operaciones críticas como pagos.

### 📌 El flujo esperado

1. El usuario añade productos al carrito
2. El sistema verifica el crédito disponible
3. Se realiza el pago
4. Se confirma la compra

### 📌 El fallo

El sistema:

* Realiza la validación en una única petición de confirmación
* No vuelve a validar el estado del carrito
* Permite reutilizar esa petición

Esto provoca que:

* Una validación válida en un contexto pueda reutilizarse en otro
* Se puedan comprar productos sin haber sido pagados

### 📌 Impacto

Esto permite:

* Bypass de validaciones de pago
* Compra de productos sin crédito
* Manipulación del flujo de compra



## 📝 Práctica

### 1️⃣ Analizar el flujo de compra

Nos logueamos con las credenciales proporcionadas.

Compramos un producto barato para entender el funcionamiento.

Observamos que durante el proceso aparece una petición clave:

* Una petición GET que parece confirmar la compra

### 2️⃣ Identificar la petición crítica

Interceptamos esa petición de confirmación.

La enviamos a Repeater.

Esta petición es la que:

* Valida que el pago se ha realizado correctamente



### 3️⃣ Preparar el exploit

Añadimos al carrito:

* La **Lightweight l33t leather jacket**

No tenemos crédito suficiente, por lo que normalmente la compra fallaría.



### 4️⃣ Reutilizar la validación

Volvemos a enviar desde Repeater la petición de confirmación previamente capturada.

Observamos que:

* El servidor acepta la solicitud
* No valida el contenido actual del carrito
* No comprueba el crédito disponible



### 5️⃣ Verificar el resultado

La compra se completa correctamente.

La chaqueta aparece como adquirida a pesar de no tener fondos suficientes.
