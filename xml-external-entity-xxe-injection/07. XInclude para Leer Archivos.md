# 🗄️ XInclude para leer archivos

## 📄 Descripción del laboratorio

Este laboratorio utiliza nuevamente la funcionalidad **Check stock**, pero con una diferencia importante respecto a los anteriores.

El servidor **construye el documento XML completo en el backend** y únicamente inserta el valor que enviamos en el parámetro **productId** dentro de ese XML antes de procesarlo.

Esto implica varias limitaciones:

* ❌ No controlamos el **documento XML completo**.
* ❌ No podemos añadir `<!DOCTYPE>`.
* ❌ No podemos definir **entidades externas ni DTD**.

El objetivo consiste en **inyectar una instrucción XInclude dentro del campo controlado**, para que el parser XML incluya el contenido del archivo:

```
/etc/passwd
```

y lo refleje directamente en la respuesta.



## 📚 Teoría

Cuando no es posible explotar **XXE tradicional**, una técnica alternativa muy común es utilizar **XInclude**.

### 📌 Qué es XInclude

**XInclude** es un estándar XML que permite incluir contenido externo dentro de un documento XML **sin necesidad de utilizar DTD ni entidades externas**.

Su sintaxis básica es:

```xml
<xi:include 
  href="ruta"
  parse="text"
  xmlns:xi="http://www.w3.org/2001/XInclude"/>
```

### 📌 Componentes clave

* **href**

Indica la ruta del recurso que queremos incluir, por ejemplo:

```
file:///etc/passwd
```

* **parse="text"**

Indica que el contenido debe tratarse como **texto plano**.

Esto es importante porque archivos como `/etc/passwd` **no son XML válido**.

* **xmlns:xi**

Declara el **namespace** necesario para que el parser reconozca la instrucción XInclude.

### 📌 Por qué funciona en este laboratorio

El ataque es posible porque:

* El input del usuario se **inserta directamente dentro del XML** generado por el backend.
* El parser XML tiene **soporte XInclude activado**.
* El contenido procesado se **refleja en la respuesta**.
* No existe **sanitización del contenido XML insertado**.

Esto permite que, aunque **XXE esté bloqueado**, podamos incluir archivos locales utilizando XInclude.

### 2️⃣ Inyección de XInclude

Dado que no podemos modificar el `DOCTYPE`, debemos realizar la inyección directamente desde el campo controlado.

Usamos el siguiente payload como valor de **productId**:

```xml
<xi:include href="file:///etc/passwd" parse="text" xmlns:xi="http://www.w3.org/2001/XInclude"/>
```

La petición queda similar a:

```xml
productId=<xi:include href="file:///etc/passwd" parse="text" xmlns:xi="http://www.w3.org/2001/XInclude"/>&storeId=1
```

Enviamos la petición al servidor.

### 3️⃣ Lectura del archivo

La respuesta del servidor cambia y ahora incluye el contenido completo del archivo:

```
/etc/passwd
```

Esto ocurre porque el parser XML:

1. Procesa el XML construido por el backend.
2. Reconoce la instrucción **XInclude**.
3. Lee el archivo local indicado.
4. Inserta su contenido dentro del documento XML.
5. Devuelve el resultado al cliente.
